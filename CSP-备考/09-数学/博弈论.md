# 博弈论

## 核心思想

博弈论研究双方最优策略下的胜负判定。CSP 主要考察：Nim 游戏、SG 函数、巴什博弈。

两个基本概念：
- **必胜态 (N-position)**：当前局面，先手有必胜策略
- **必败态 (P-position)**：当前局面，无论先手怎么走，后手都有必胜策略

## Nim 游戏

n 堆石子，每堆 a[i] 个。两人轮流从任意一堆取任意正数个石子。无法操作者输。

**Nim 和**：`X = a1 ⊕ a2 ⊕ ... ⊕ an`（所有堆数量的异或）

**结论**：X ≠ 0 → 先手必胜，X = 0 → 先手必败。

```cpp
bool nimWin(vector<int>& piles) {
    int x = 0;
    for (int p : piles) x ^= p;
    return x != 0;
}
```

**必败态变必胜态的策略**：若 `(a[i] ^ x) < a[i]`，则从第 i 堆取 `a[i] - (a[i] ^ x)` 个石子。

## SG 函数（Sprague-Grundy）

将任意公平组合游戏转化为 Nim 游戏。

定义：
- `SG(x) = mex{SG(y) | y 是 x 的后继状态}`，其中 mex = 最小未出现的非负整数
- 必败态等价于 SG(x) = 0

**SG 定理**：由 n 个子游戏组成的组合游戏，`SG(总) = SG(1) ⊕ SG(2) ⊕ ... ⊕ SG(n)`。

总游戏必胜 ⇔ SG(总) ≠ 0。

```cpp
// 计算有向图游戏的SG值
int sg[MAXN];

int getSG(int x) {
    if (sg[x] != -1) return sg[x];
    set<int> st;
    for (int y : nextStates[x]) {
        st.insert(getSG(y));
    }
    for (int i = 0; ; i++)
        if (!st.count(i)) return sg[x] = i;
}
```

## 巴什博弈

一堆 n 个石子，每次取 1~m 个。无法取者输。

**结论**：若 `n % (m+1) != 0` 则先手必胜（先手取 `n % (m+1)` 个即可）。

```cpp
bool bashWin(int n, int m) {
    return n % (m + 1) != 0;
}
```

## 威佐夫博弈

两堆石子，每次可以从一堆取任意个，或同时从两堆取相同数量。

**结论**：必败态是奇异局势 `(a_k, b_k)`，其中：
- `a_k = floor(k × φ)`, `b_k = a_k + k`, `φ = (1+√5)/2`（黄金分割比）

```cpp
bool wythoff(int a, int b) {
    if (a > b) swap(a, b);
    int k = b - a;
    int ak = (int)(k * (1.0 + sqrt(5.0)) / 2);
    return a != ak;  // true = 先手胜
}
```

## 经典例题：取石子加强版

> 一堆 n 个石子，每次可取质数个。判断先手胜负。

打 SG 表：

```cpp
vector<int> primes = sieve(1000);  // 最小质数
int sg[10001];
memset(sg, -1, sizeof(sg));

int getSG(int n) {
    if (sg[n] != -1) return sg[n];
    set<int> st;
    for (int p : primes) {
        if (p > n) break;
        st.insert(getSG(n - p));
    }
    for (int i = 0; ; i++)
        if (!st.count(i)) return sg[n] = i;
}
```

## 必胜态与必败态的递推关系

对于任意游戏，定义：
- 一个状态是必败态，当且仅当它的所有后继都是必胜态
- 一个状态是必胜态，当且仅当存在至少一个后继是必败态

这可用于没有 SG 公式的游戏，通过 DP 递推。

## ⚠️ 易错点

1. **SG 函数记忆化**：忘记 `memset(sg, -1)` 导致重复计算超时
2. **SG 值很大**：SG 可能超过游戏状态数，数组要开够
3. **Nim 和 0 是必败**：不是必胜，不要记反
4. **巴什博弈的 m+1**：不是 m，而是 m+1 为一个循环周期
5. **子游戏混淆**：SG 定理要求子游戏相互独立，有交互的不能直接用
6. **威佐夫博弈精度**：√5 计算容易精度丢失，用整数比较可避免
