# 字符串哈希 (String Hashing)

## 核心思想

将字符串映射为一个整数，通过比较哈希值来判断字符串是否相等。类似给每个字符串分配一个"数字指纹"，O(1) 比较两字符串（在哈希冲突概率极低的前提下）。

**多项式滚动哈希**是最常用的方法：

$$hash(S) = \sum_{i=0}^{n-1} S[i] \times base^{n-1-i} \pmod{mod}$$

递推形式：`hash = (hash * base + s[i]) % mod`

### 为什么选 131/13331？

- base 取大于字符集大小的质数，131 和 13331 经验上冲突率极低
- mod 用 $2^{64}$（`unsigned long long` 自然溢出），省去取模运算，速度最快

## 子串哈希 O(1) 查询

预处理前缀哈希 `h[i]` 和 base 幂次 `p[i]`：

$$
hash(s[l..r]) = h[r+1] - h[l] \times p[r-l+1]
$$

类比：把 s[0..r] 的哈希值减去 s[0..l-1] 的贡献（左移相应位数）。

## C++ 模板

```cpp
#include <bits/stdc++.h>
using namespace std;
using ull = unsigned long long;

const ull BASE = 131;

struct StringHash {
    vector<ull> h, p;
    int n;

    StringHash(const string &s) {
        n = s.size();
        h.resize(n + 1, 0);
        p.resize(n + 1, 1);
        for (int i = 0; i < n; i++) {
            h[i + 1] = h[i] * BASE + s[i];
            p[i + 1] = p[i] * BASE;
        }
    }

    // 获取子串 s[l..r] 的哈希值 (0-indexed, 闭区间)
    ull get(int l, int r) {
        return h[r + 1] - h[l] * p[r - l + 1];
    }
};
```

### 双哈希（防冲突）

```cpp
using ull = unsigned long long;
const ull BASE1 = 131, BASE2 = 13331;

struct DoubleHash {
    vector<ull> h1, h2, p1, p2;
    int n;

    DoubleHash(const string &s) {
        n = s.size();
        h1.resize(n + 1); h2.resize(n + 1);
        p1.resize(n + 1, 1); p2.resize(n + 1, 1);
        for (int i = 0; i < n; i++) {
            h1[i + 1] = h1[i] * BASE1 + s[i];
            h2[i + 1] = h2[i] * BASE2 + s[i];
            p1[i + 1] = p1[i] * BASE1;
            p2[i + 1] = p2[i] * BASE2;
        }
    }

    pair<ull, ull> get(int l, int r) {
        return {
            h1[r + 1] - h1[l] * p1[r - l + 1],
            h2[r + 1] - h2[l] * p2[r - l + 1]
        };
    }
};
```

## 应用场景

### 1. 字符串匹配 (Rabin-Karp)

模式串 P 长度 m，文本串 T 长度 n。O(n+m)：算 P 的哈希，滑动窗口算 T 中每个长度为 m 的子串哈希，比较。

### 2. 最长公共前缀 (LCP)

二分答案 + 哈希检查：在 O(log n) 时间内找到两个后缀的最长公共前缀。

```cpp
int lcp(StringHash &h, int i, int j) {
    int lo = 0, hi = min(h.n - i, h.n - j);
    while (lo < hi) {
        int mid = (lo + hi + 1) / 2;
        if (h.get(i, i + mid - 1) == h.get(j, j + mid - 1))
            lo = mid;
        else
            hi = mid - 1;
    }
    return lo;
}
```

### 3. 判断回文

正着算一次哈希，反着算一次哈希，比较正反子串哈希。

## 复杂度

| 操作 | 时间复杂度 |
|------|-----------|
| 预处理 | O(n) |
| 子串哈希 | O(1) |
| 字符串匹配 | O(n+m) |
| LCP (二分) | O(log n) |

## CSP 真题场景

- **J2022 T3** 逻辑表达式 — 可用哈希做部分字符串判断
- **S2021 T2** 括号序列 — 区间 DP 配合哈希判回文
- 各种字符串匹配型题目的替代解

## ⚠️ 易错点

1. **索引混淆**：`get(l, r)` 中 `h[r+1]` 而非 `h[r]`，多 1 位是前缀哈希的惯例
2. **溢出后 mod 不为 $2^{64}$**：用 `ull` 自然溢出才等价于 mod $2^{64}$，不要用 `int`
3. **base 太小**：base 必须大于字符集大小（如只有小写字母也要 > 26）
4. **生日悖论**：单哈希在 $10^5$ 级别有较小碰撞风险，S 组建议双哈希
5. **`get` 中减法**：`ull` 自然溢出减法正确是因为无符号整数的循环性质
