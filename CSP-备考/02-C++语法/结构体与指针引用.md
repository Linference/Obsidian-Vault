# 结构体与指针引用

## 一、结构体（struct）

### 定义与使用

```cpp
struct Student {
    string name;
    int age;
    double score;
};  // ⚠️ 别忘了分号

int main() {
    Student s1;                    // C++ 中 struct 关键字可省略
    s1.name = "Zhang San";
    s1.age = 16;
    s1.score = 95.5;

    Student s2 = {"Li Si", 17, 88.0};  // 聚合初始化

    cout << s1.name << " " << s1.age << endl;
}
```

### 结构体成员访问

- 通过对象：`.`（点运算符）
- 通过指针：`->`（箭头运算符）

```cpp
Student s;
Student *p = &s;
s.name = "Alice";   // 通过对象
p->name = "Bob";    // 通过指针，等价 (*p).name
```

### 结构体数组与嵌套

```cpp
struct Point {
    int x, y;
};

struct Rectangle {
    Point topLeft;    // 嵌套结构体
    Point bottomRight;
    int area() {      // 结构体内可以定义函数！（C++ 中的 struct 几乎等于 class）
        return (bottomRight.x - topLeft.x) * (bottomRight.y - topLeft.y);
    }
};
```

## 二、结构体的内存对齐

### 什么是内存对齐

CPU 访问内存时，如果数据地址是其大小的整数倍，访问效率最高。编译器为此会在结构体成员之间插入**填充字节（padding）**。

```cpp
struct A {
    char  c;   // 1 字节 → 偏移 0
    int   i;   // 4 字节 → 编译器在 c 后补 3 字节，使得 i 在偏移 4
    short s;   // 2 字节 → 偏移 8
};
// sizeof(A) = 12，不是 1+4+2=7！
// 结构体末尾可能也要对齐到最大成员的大小（这里是 4 的倍数）
```

### 对齐规则（简化版）

1. 每个成员的偏移量必须是自身大小的整数倍。
2. 结构体总大小必须是最大成员大小的整数倍。

```cpp
struct B {
    char  a;  // 偏移 0，占 1 字节
    short b;  // 偏移 2（不是 1！补了 1 字节）
    int   c;  // 偏移 4，占 4 字节
    char  d;  // 偏移 8，占 1 字节，末尾补 3 字节
};
// sizeof(B) = 12（对齐到 int 的 4 字节边界）

struct C {
    int   c;  // 偏移 0
    short b;  // 偏移 4
    char  a;  // 偏移 6
    char  d;  // 偏移 7，末尾补 0 字节（最大成员 int=4，8 已是 4 的倍数）
};
// sizeof(C) = 8  ← 看！调整成员位置可以缩小结构体！节省 33%！
```

> **CSP 高频**：给一个结构体定义，问 `sizeof` 的值。你必须画偏移量表。

### 如何避免对齐浪费

- 把占用大的成员放前面（`double > long long > int > short > char`）
- 或者用 `#pragma pack(1)` 取消对齐（但会降低性能）

## 三、指针

### 基本概念

**指针** = 存放另一个变量**地址**的变量。

```cpp
int x = 42;
int *p = &x;    // p 存储 x 的地址
cout << p;      // 输出 x 的地址（如 0x7fff...）
cout << *p;     // 解引用，输出 42
*p = 100;       // 通过指针修改 x，现在 x = 100
```

### 指针运算

```cpp
int arr[5] = {10, 20, 30, 40, 50};
int *p = arr;     // p 指向 arr[0]，arr 本身是首地址
int *q = arr + 2; // q 指向 arr[2]

cout << *(p + 1); // 20（arr[1]）
cout << q - p;    // 2（相差 2 个 int 的距离，不是 8 字节！）
// 指针加减的步长 = sizeof(指向类型)
```

### 指针与 const

```cpp
int x = 10, y = 20;

const int *p1 = &x;   // 指向常量的指针：不能通过 p1 修改值，但 p1 可以指向别处
// *p1 = 30;  ❌ 错误
p1 = &y;      // ✅ 可以

int *const p2 = &x;   // 常量指针：p2 始终指向 x，不能改指向
*p2 = 30;             // ✅ 可以修改值
// p2 = &y;  ❌ 错误

const int *const p3 = &x;  // 什么都改不了
```

### new / delete 动态内存分配

```cpp
int *p = new int;       // 分配一个 int
int *arr = new int[100]; // 分配 100 个 int 的数组
*p = 42;
arr[0] = 1;

delete p;               // 释放单个对象
delete[] arr;           // 释放数组（必须用 delete[]！）
// 忘记 delete → 内存泄漏
// delete 后指针变成"悬空指针"，应设为 nullptr
p = nullptr;
```

> **CSP 重要**：`new` 和 `delete` 配对；`new[]` 和 `delete[]` 配对。混用是未定义行为。

### 空指针与野指针

```cpp
int *p1 = nullptr;      // C++11 空指针，推荐
int *p2 = NULL;         // 传统写法，本质是 0
int *p3 = 0;            // 也可以，但不推荐

int *wild;              // ❌ 野指针！未初始化，指向随机地址
*wild = 5;              // ❌ 未定义行为，可能崩溃
```

## 四、引用

### 引用的本质

**引用是变量的别名**。创建时必须初始化，之后不能改变指向（绑定终身）。

```cpp
int x = 10;
int &ref = x;   // ref 是 x 的引用（别名）
ref = 20;       // 等价于 x = 20
cout << x;      // 输出 20
```

### 引用 vs 指针

| 特性 | 引用 | 指针 |
|------|------|------|
| 必须初始化 | ✅ 必须（定义时绑定） | ❌ 可以延后 |
| 能否为空 | ❌ 不能（没有"空引用"） | ✅ nullptr |
| 修改指向 | ❌ 不能（终身绑定） | ✅ 可以 |
| 语法 | 简洁（`ref` 直接用） | 需要 `*` 和 `&` |
| 底层实现 | 通常也是指针（编译器优化） | 指针变量 |

> **什么时候用引用**：函数参数（避免拷贝）、返回值（链式调用如 `cout`）。  
> **什么时候用指针**：需要表示"不存在"（nullptr）、动态分配、链表/树等数据结构。

## 五、链表节点示例（指针经典应用）

```cpp
struct Node {
    int data;
    Node *next;          // 指向下一个节点的指针
    Node(int val) : data(val), next(nullptr) {}  // 构造函数
};

// 创建链表 1 → 2 → 3
Node *head = new Node(1);
head->next = new Node(2);
head->next->next = new Node(3);

// 遍历链表
Node *p = head;
while (p != nullptr) {
    cout << p->data << " ";
    p = p->next;
}

// 释放链表（防止内存泄漏）
while (head != nullptr) {
    Node *t = head;
    head = head->next;
    delete t;
}
```

## ⚠️ 易错点

1. **`sizeof` 结构体 ≠ 成员大小之和**——内存对齐导致实际大小更大。
2. **`delete` 和 `delete[]` 不能混用**——`new[]` 出来的必须用 `delete[]` 释放。
3. **引用声明时必须初始化**——`int &ref;` 编译错误。
4. **指针解引用前检查是否为 nullptr**——对空指针解引用是未定义行为。
5. **函数返回局部变量的引用/指针**——局部变量在函数返回后销毁，返回其引用/指针是悬空引用/野指针。
6. **结构体定义末尾有分号**——忘了编译器报错信息可能很奇怪。
7. **`p++` 移动的字节数是 `sizeof(*p)`**——不是 1 字节。
8. **指针和数组名的区别**——`sizeof(指针)` 是 8 字节（64 位），`sizeof(数组名)` 是整个数组大小。
9. **引用底层通常是指针**——但这不影响语义。不要试图对引用做指针运算。
