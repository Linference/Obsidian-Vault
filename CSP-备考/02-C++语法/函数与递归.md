# 函数与递归

## 一、函数基础

### 什么是函数

函数是**封装了特定功能的代码块**，接收输入（参数），返回输出（返回值）。目的是**复用代码、降低复杂度、提高可读性**。

### 函数定义

```cpp
返回类型 函数名(形参列表) {
    // 函数体
    return 返回值;  // 如果返回类型不是 void
}
```

```cpp
int add(int a, int b) {
    return a + b;
}

void printSum(int a, int b) {  // void 表示无返回值
    cout << a + b << endl;
    // 可以写 return; 但不返回值
}
```

### 函数声明与定义分离

```cpp
// 声明（通常在 .h 文件或 main 之前）
int add(int a, int b);

// main
int main() {
    cout << add(3, 5);
    return 0;
}

// 定义（可以在 main 之后）
int add(int a, int b) {
    return a + b;
}
```

## 二、参数传递

### 值传递（Pass by Value）

```cpp
void swap(int a, int b) {  // ❌ 不会影响实参
    int t = a; a = b; b = t;
}
// 调用 swap(x, y) 后，x 和 y 的值不变！
// 因为 a 和 b 是 x 和 y 的副本
```

### 引用传递（Pass by Reference）

```cpp
void swap(int &a, int &b) {  // ✅ & 表示引用，操作的是实参本身
    int t = a; a = b; b = t;
}
// 调用 swap(x, y) 后，x 和 y 交换成功
```

> **为什么用引用**：
> 1. 需要修改实参（如上例）
> 2. 避免大对象拷贝（如 `void print(const vector<int> &v)` —— 加 const 保证不修改，同时避免整个 vector 的拷贝）

### 指针传递

```cpp
void swap(int *a, int *b) {
    int t = *a; *a = *b; *b = t;
}
// 调用：swap(&x, &y);
```

### 传递方式对比

| 方式 | 语法 | 修改实参 | 拷贝开销 | 推荐场景 |
|------|------|:------:|:------:|----------|
| 值传递 | `int x` | ❌ | 有（小对象可忽略） | 基本类型，不需要修改 |
| 引用传递 | `int &x` | ✅ | 无 | 需要修改实参 |
| const 引用 | `const int &x` | ❌（保护） | 无 | 大对象，只读 |
| 指针传递 | `int *x` | ✅ | 无（只拷贝指针） | C 风格，可接受 nullptr |

## 三、默认参数

```cpp
int power(int base, int exp = 2) {  // exp 有默认值 2
    int result = 1;
    for (int i = 0; i < exp; i++) result *= base;
    return result;
}

power(5);      // 5² = 25（exp 用默认值 2）
power(5, 3);   // 5³ = 125
```

> **规则**：默认参数必须从右向左连续设置。`void f(int a = 1, int b)` ❌ 非法！

## 四、函数重载

**同一作用域内，函数名相同但参数列表不同**（个数、类型、顺序）。

```cpp
int max(int a, int b)          { return a > b ? a : b; }
double max(double a, double b) { return a > b ? a : b; }
int max(int a, int b, int c)   { return max(max(a, b), c); }
```

> **注意**：重载只看**参数列表**，不看返回类型。`int f()` 和 `void f()` 不能构成重载。

## 五、递归

### 什么是递归

函数**直接或间接调用自身**。用于解决具有**自相似结构**的问题。

### 递归三要素（必须全部满足！）

1. **终止条件（Base Case）**：递归的出口，到什么时候停下来。
2. **缩小问题规模**：每次递归调用必须朝终止条件靠近。
3. **调用自身**：用更小的子问题的解构造当前问题的解。

### 经典示例：阶乘

```cpp
int factorial(int n) {
    if (n == 0) return 1;           // 终止条件
    return n * factorial(n - 1);   // 缩小规模 + 调用自身
}
// factorial(5) = 5 * factorial(4)
//              = 5 * 4 * factorial(3)
//              = 5 * 4 * 3 * factorial(2)
//              = 5 * 4 * 3 * 2 * factorial(1)
//              = 5 * 4 * 3 * 2 * 1 * factorial(0)
//              = 5 * 4 * 3 * 2 * 1 * 1 = 120
```

### 斐波那契数列（含记忆化）

```cpp
// 普通递归——O(2^n) 指数复杂度
int fib(int n) {
    if (n <= 1) return n;          // 终止条件
    return fib(n - 1) + fib(n - 2); // 大量重复计算！
}

// 记忆化递归——O(n)
const int MAXN = 100;
long long memo[MAXN];

long long fib_memo(int n) {
    if (n <= 1) return n;
    if (memo[n] != 0) return memo[n];   // 已经算过，直接返回
    return memo[n] = fib_memo(n - 1) + fib_memo(n - 2); // 算过就存
}
```

### 递归调用栈与栈溢出

每次递归调用都会在**调用栈**上分配一帧（存储局部变量、返回地址等）。递归层次太深会导致**栈溢出（Stack Overflow）**。

```cpp
// ❌ 危险：n=100000 时栈溢出
void dfs(int n) {
    if (n == 0) return;
    dfs(n - 1);  // 100000 层调用
}
```

> **经验值**：栈大小通常 8MB，每层递归约占用几十到几百字节。一般递归深度不超过 **10⁵** 级别。CSP 题中如果数据规模超过 10⁴，考虑用迭代或手动栈。

### 尾递归与编译器优化

```cpp
// 普通递归（每次需要在栈上保留 n 的值）
int sum(int n) {
    if (n == 0) return 0;
    return n + sum(n - 1);  // 返回后还要加 n，所以必须保留栈帧
}

// 尾递归（返回语句只有递归调用本身，不带额外运算）
int sum_tail(int n, int acc = 0) {
    if (n == 0) return acc;
    return sum_tail(n - 1, acc + n);  // 尾递归，编译器可能优化为迭代
}
```

> 尾递归优化：编译器识别尾递归后，直接复用当前栈帧，不额外消耗栈空间。但 C++ 标准不保证，取决于编译器和优化等级。

## 六、inline 函数

```cpp
inline int square(int x) {
    return x * x;
}
// 编译器会将 square(5) 展开为 5*5，省去函数调用开销
// 适合短小频繁调用的函数
// 注意：inline 只是建议，编译器可以忽略
```

## ⚠️ 易错点

1. **递归缺少终止条件**——死递归 → 栈溢出。
2. **值传递不会修改实参**——swap 函数忘记写 `&` 是最经典的错误。
3. **递归未记忆化导致指数复杂度**——斐波那契、爬楼梯等题目直接用朴素递归会超时。
4. **大数组定义在函数内部**——局部变量存在栈上，`int arr[1000000]` 直接栈溢出。大数组应定义为全局或 static。
5. **函数声明末尾要有分号**，定义末尾没有分号。
6. **默认参数只能在声明或定义中出现一次**——通常在声明中写默认值。
7. **递归中全局变量污染**——多次调用间全局变量没清零，导致结果错误。
