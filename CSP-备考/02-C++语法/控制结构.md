# 控制结构

## 一、if / else if / else

最基本的条件判断结构。

```cpp
if (条件1) {
    // 条件1为真时执行
} else if (条件2) {
    // 条件1为假 且 条件2为真时执行
} else {
    // 以上条件都为假时执行
}
```

### 悬空 else 问题

```cpp
int x = 10, y = 5;
if (x > 0)
    if (y > 10)
        cout << "A";
else
    cout << "B";  // ⚠️ 这个 else 属于最近的 if (y > 10)，不是 if (x > 0)!
// 输出：B（因为 x>0 为真进入内层，y>10 为假，else 执行）
```

> **规则**：else 总是与最近的未配对 if 配对。用大括号 `{}` 消除歧义。

### 多条件判断

```cpp
// 判断闰年：能被4整除 且 不能被100整除，或能被400整除
if ((year % 4 == 0 && year % 100 != 0) || year % 400 == 0) {
    cout << "闰年";
}
```

## 二、switch-case

用于对同一个变量（整数或枚举）的多个离散值进行分支。比多重 if-else 可读性更好，且编译器可能优化为跳转表。

```cpp
switch (表达式) {  // 表达式必须是整型或枚举类型（char, int, enum ...）
    case 常量1:
        语句;
        break;     // ⚠️ 没有 break 会"穿透"到下一个 case！
    case 常量2:
        语句;
        break;
    default:
        语句;      // 所有 case 都不匹配时执行
        break;
}
```

### switch 陷阱

```cpp
int score = 85, level;
switch (score / 10) {
    case 10:
    case 9:  level = 'A'; break;  // ✅ 利用了穿透：9和10都是A
    case 8:  level = 'B'; break;
    case 7:  level = 'C'; break;
    case 6:  level = 'D'; break;
    default: level = 'F'; break;
}
```

> **CSP 坑**：忘记写 `break` 导致 case 穿透，是所有初学者的必修课。

### switch 限制

- 表达式只能是整数、字符、枚举，**不能是字符串或浮点数**
- case 后面必须是**编译期常量**，不能是变量
- 不同 case 的值不能相同

```cpp
// ❌ 错误示例：
string s = "abc";
switch (s) { ... }  // ❌ 不能是 string

int x = 5;
switch (x) {
    case x: ...     // ❌ case 后面不能是变量
}

switch (3.14) {     // ❌ 不能是浮点数
    ...
}
```

## 三、for 循环

```cpp
for (初始化; 条件; 更新) {
    循环体;
}
// 执行顺序：
// 1. 初始化（只执行一次）
// 2. 判断条件 → false 则退出
// 3. 执行循环体
// 4. 执行更新
// 5. 回到步骤2
```

```cpp
// 遍历 0 到 n-1
for (int i = 0; i < n; i++) { ... }

// 从 n 倒序到 1
for (int i = n; i >= 1; i--) { ... }

// 多个变量
for (int i = 0, j = n - 1; i < j; i++, j--) { ... }

// 死循环
for (;;) { ... }  // 三部分都可以省略
```

### for 中定义的变量作用域

```cpp
for (int i = 0; i < n; i++) { ... }
cout << i;  // ❌ i 的作用域仅限于 for 循环内（C++ 标准）

int i;
for (i = 0; i < n; i++) { ... }
cout << i;  // ✅ i = n，此时 i 在循环外也可见
```

## 四、while 循环

### while（先判断，后执行）

```cpp
while (条件) {
    // 条件为真时反复执行
    // 循环体内应有改变条件的语句，否则死循环
}
```

### do-while（先执行，后判断）

```cpp
do {
    // 先执行一次循环体
    // 然后判断条件
} while (条件);  // ⚠️ 注意这里有分号！
```

```cpp
// 区别演示
int x = 10;
while (x < 5) { cout << "A"; }  // 不执行（条件一开始就不满足）
do { cout << "B"; } while (x < 5); // 执行一次，输出 B
```

### while 输入读取

```cpp
int x;
while (cin >> x) {  // 读到文件末尾或输入错误时退出
    cout << x * 2 << endl;
}
// cin >> x 返回 cin 本身，转为 bool，EOF 或失败时为 false
```

## 五、break 与 continue

| 语句 | 作用 | 影响范围 |
|:----:|------|----------|
| `break` | **跳出**当前循环 / switch | 最近的一层 |
| `continue` | **跳过**本次迭代的剩余部分，进入下一次迭代 | 最近的循环 |

```cpp
// break 示例
for (int i = 1; i <= 10; i++) {
    if (i == 5) break;  // 到 i=5 时退出循环
    cout << i << " ";   // 输出：1 2 3 4
}

// continue 示例
for (int i = 1; i <= 10; i++) {
    if (i % 2 == 0) continue;  // 跳过偶数
    cout << i << " ";          // 输出：1 3 5 7 9
}
```

### 嵌套循环中的 break

```cpp
for (int i = 0; i < 3; i++) {
    for (int j = 0; j < 3; j++) {
        if (j == 1) break;  // ⚠️ 只跳出内层 j 循环，外层 i 循环继续
        cout << i << "," << j << " ";
    }
}
// 输出：0,0 1,0 2,0
```

## 六、goto（了解，别用）

```cpp
int i = 0;
loop:                    // 标签
    if (i < 5) {
        cout << i << " ";
        i++;
        goto loop;       // 跳转到标签
    }
// 输出：0 1 2 3 4
```

> **CSP 观点**：goto 使程序控制流混乱，几乎总能用循环替代。初赛偶尔出个 goto 题考你"这段代码输出什么"，但实际编程**不要用**。

## ⚠️ 易错点

1. **if 后面没加大括号**——只有紧跟的第一条语句属于 if，缩进是骗人的。
2. **悬空 else**——else 配最近的未配对 if。
3. **switch 忘记 break**——穿透是特性也是陷阱。
4. **`=` 和 `==` 混淆**——`if (x = 5)` 永远为 true；使用 Yoda 写法 `if (5 == x)` 可防（写错 `if (5 = x)` 编译期报错）。
5. **for 的 i 作用域**——C++ 标准中 for 内定义的 i 在循环外不可见（老编译器可能有差异）。
6. **do-while 后面的分号**——`while (cond);` 别忘了分号，这是最容易的语法错误。
7. **continue 跳到"更新"前**——在 while 中，continue 直接跳到条件判断，小心跳过增量语句导致死循环。
8. **break 只跳出一层**——嵌套循环中想全跳出需要用标志变量或 goto。
