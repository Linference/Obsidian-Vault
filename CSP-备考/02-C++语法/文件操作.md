# 文件操作

## 一、为什么需要文件操作

CSP 复赛的评测系统是**用文件输入输出**的：你的程序需要从 `problem.in` 读取数据，输出到 `problem.out`。如果你写了 `cin/cout` 交互式输入输出而没有用文件重定向，**即使算法完全正确也是 0 分**。

> 这是每年复赛大量 0 分的头号原因——不是不会做，是没写文件操作。

## 二、freopen —— CSP 生存必备

### 标准写法（贴到 main 函数最开头）

```cpp
#include <cstdio>  // freopen 需要

int main() {
    freopen("problem.in", "r", stdin);   // 将 stdin 重定向到输入文件
    freopen("problem.out", "w", stdout);  // 将 stdout 重定向到输出文件

    // 从这里开始，cin / scanf 自动从文件读，cout / printf 自动写到文件
    int n;
    cin >> n;
    cout << n * 2 << endl;

    fclose(stdin);   // 好习惯：关闭文件
    fclose(stdout);
    return 0;
}
```

### 参数说明

```cpp
freopen("文件名", "模式", 流);
// 模式：
//   "r"  → 只读（文件必须存在）
//   "w"  → 只写（文件不存在则创建，存在则清空）
//   "a"  → 追加写（在文件末尾添加）
//   "r+" → 读写
// 流：
//   stdin  → 标准输入
//   stdout → 标准输出
//   stderr → 标准错误输出
```

### 本地调试技巧

```cpp
// 用宏控制在本地用终端，提交时用文件
#ifdef LOCAL
    // 本地：不重定向，直接在终端输入输出
#else
    freopen("problem.in", "r", stdin);
    freopen("problem.out", "w", stdout);
#endif

// 编译时：g++ -DLOCAL main.cpp -o main
// 提交时不要定义 LOCAL 宏
```

## 三、C++ 文件流（fstream）

### ifstream（读文件）

```cpp
#include <fstream>

ifstream fin("data.in");  // 打开文件
if (!fin.is_open()) {     // ⚠️ 检查是否成功打开
    cerr << "Failed to open file!" << endl;
    return 1;
}

int n;
fin >> n;                 // 从文件读取，用法和 cin 一样
// 或用 fin.getline(buf, size) 读一行

fin.close();              // 关闭文件（析构函数也会自动关闭）
```

### ofstream（写文件）

```cpp
#include <fstream>

ofstream fout("data.out");  // 默认 ios::out，会清空已有内容
// ofstream fout("data.out", ios::app);  // 追加模式

fout << "Hello CSP!" << endl;  // 写入文件

fout.close();
```

### 文件打开模式

| 模式 | 含义 |
|------|------|
| `ios::in` | 打开用于读取 |
| `ios::out` | 打开用于写入（清空已有内容） |
| `ios::app` | 追加模式（写入追加到文件末尾） |
| `ios::binary` | 二进制模式 |
| `ios::ate` | 打开后定位到文件末尾 |
| `ios::trunc` | 如果文件存在，清空内容 |

```cpp
// 组合模式用 | 连接
ofstream fout("file.txt", ios::out | ios::app);  // 追加写
ifstream fin("file.txt", ios::in | ios::binary); // 二进制读
```

## 四、读取到文件末尾（EOF）

### cin 的 EOF 判断

```cpp
// 方法一：while (cin >> x)
int x;
while (cin >> x) {  // 读到文件末尾或输入错误时退出
    cout << x * x << endl;
}

// 方法二：判断 EOF
int x;
while (scanf("%d", &x) != EOF) {  // C 风格
    printf("%d\n", x * x);
}

// 方法三：cin.eof()
int x;
while (true) {
    cin >> x;
    if (cin.eof()) break;  // 检查是否到达文件末尾
    cout << x << endl;
}
```

### 读取不定数量的行

```cpp
string line;
while (getline(cin, line)) {  // 逐行读取直到 EOF
    cout << "Read: " << line << endl;
}
```

## 五、getline 的注意事项

### getline 与 cin 混用的坑

```cpp
int n;
cin >> n;                    // 读入整数，但换行符 '\n' 留在缓冲区！

string line;
getline(cin, line);           // ⚠️ 读到的是一行空行（只有换行符）！

// 正确做法：在读整数后清理缓冲区
cin.ignore();  // 或 cin.ignore(numeric_limits<streamsize>::max(), '\n');
getline(cin, line);           // ✅ 现在读到真正的下一行
```

### getline 的两个版本

```cpp
// C++ string 版（#include <string>）
string s;
getline(cin, s);              // 读一行到 string
getline(cin, s, ',');         // 以逗号为分隔符

// C 风格版（#include <cstdio> 或 <iostream>）
char buf[1000];
cin.getline(buf, 1000);       // 读最多 999 个字符（留 '\0' 位置）
cin.getline(buf, 1000, ',');  // 指定分隔符
```

## 六、文件判断是否成功打开

```cpp
// 方法一：is_open()
ifstream fin("data.in");
if (!fin.is_open()) {
    cerr << "Cannot open file!" << endl;
    return 1;
}

// 方法二：直接判断流状态
if (!fin) { ... }

// 方法三：ifstream 构造函数 + 条件判断
ifstream fin;
fin.open("data.in");
if (fin.fail()) { ... }

// freopen 的判断
if (freopen("data.in", "r", stdin) == NULL) {
    cerr << "Cannot open file!" << endl;
    return 1;
}
```

## ⚠️ 易错点

1. **复赛不写 freopen → 0 分**——这是最不应该丢的分。
2. **文件名写错**——通常是 `problem.in` / `problem.out`，注意题目要求的文件名（可能不是 problem）。
3. **cin 和 getline 混用**——cin 后缓冲区残留换行符，必须先 `cin.ignore()`。
4. **freopen 写到一半程序崩溃**——文件缓冲区没刷新，导致输出不完整。崩溃了 `fclose` 都来不及。可以在关键处 `fflush(stdout)`。
5. **ofstream 默认清空已有文件**——如果要保留原内容，加 `ios::app`。
6. **文件路径**——CSP 评测机上文件通常放在程序同级目录，写相对路径 `"problem.in"` 即可，别写绝对路径。
7. **close 不是强制要求**——但体现了你的良好习惯。程序正常结束时系统会帮你关，但大程序建议手动关闭释放资源。
