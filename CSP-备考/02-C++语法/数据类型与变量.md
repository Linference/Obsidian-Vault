# 数据类型与变量

## 一、基本数据类型

C++ 的内置基本类型及字节大小（以 64 位系统 / CSP 标准为准）：

| 类型 | 关键字 | 字节数 | 大致范围 | 精度 |
|------|--------|:-----:|----------|:---:|
| 布尔型 | `bool` | 1 | `true` / `false` | — |
| 字符型 | `char` | 1 | -128 ~ 127 或 0 ~ 255 | — |
| 短整型 | `short` | 2 | -32768 ~ 32767 | — |
| 整型 | `int` | **4** | **约 ±2.1×10⁹** | — |
| 长整型 | `long long` | **8** | **约 ±9.2×10¹⁸** | — |
| 单精度浮点 | `float` | 4 | 约 ±3.4×10³⁸ | **6~7 位有效数字** |
| 双精度浮点 | `double` | 8 | 约 ±1.7×10³⁰⁸ | **15~16 位有效数字** |

> **必记**：`int` 是 **4 字节**（32 位），能表示约 ±21 亿。`long long` 是 **8 字节**（64 位），能表示约 ±9×10¹⁸。
> `char` 在 C++ 中就是 1 字节整数——`'A'` 本质就是 `65`。

### 为什么需要多种整数类型

不同的数据类型决定了**内存占用**和**数值范围**。选对类型：
- 省内存（嵌入式/大数据量）
- 防溢出（大数运算用 `long long`）
- 匹配算法需求（位运算常用 `unsigned int`）

```cpp
int a = 100000;
int b = 100000;
long long c = a * b;  // ❌ a*b 先以 int 计算，溢出为负数，再转 long long
// 正确姿势：
long long c = 1LL * a * b;  // ✅ 1LL 强制整个表达式用 long long 计算
```

### 无符号类型（unsigned）

```cpp
unsigned int x = 0;
x = x - 1;  // ⚠️ x = 4294967295（UINT_MAX），不是 -1！
// 无符号整数不会为负，减法"回绕"
```

> **CSP 陷阱**：有符号与无符号混用时，有符号数被隐式转为无符号，导致 `-1 < 1u` 为 false。

## 二、浮点数

### float vs double

```cpp
float  f = 1.0 / 3;  // 精度约 7 位有效数字
double d = 1.0 / 3;  // 精度约 15 位有效数字
// float 型常量写 f 后缀：3.14f，否则默认是 double
```

### 浮点比较陷阱

```cpp
double a = 0.1 + 0.2;
if (a == 0.3) { ... }  // ❌ 可能不成立！浮点精度误差
// 正确姿势：
const double EPS = 1e-9;
if (fabs(a - 0.3) < EPS) { ... }  // ✅ 用误差范围判断
```

> **为什么 0.1+0.2 ≠ 0.3**：0.1 和 0.2 在二进制中都是无限循环小数，截断后产生舍入误差。

## 三、类型转换

### 隐式转换（自动）

```cpp
int a = 5;
double b = a;      // int → double（安全，自动）
int c = 3.14;      // double → int（截断！c = 3）
char d = 65;       // int → char（d = 'A'）
```

### 整型除法截断

```cpp
int a = 5 / 2;     // a = 2，不是 2.5！
double b = 5 / 2;  // b = 2.0（先 int 除法截断为 2，再转 double）
double c = 5.0 / 2; // c = 2.5（✅ 这里 5.0 是 double，触发隐式转换）
```

> **CSP 高频坑**：`5 / 2` 的结果是 `2`（int），不是 `2.5`。想要浮点结果，至少一个操作数是浮点型。

### 显式转换

```cpp
double pi = 3.14159;
int a = (int)pi;           // C风格，a = 3
int b = int(pi);           // 函数风格
int c = static_cast<int>(pi);  // C++ 推荐，编译期检查
```

### 常见转换方向

| 转换 | 是否安全 | 说明 |
|------|:------:|------|
| `int → long long` | ✅ 安全 | 自动扩展 |
| `int → double` | ✅ 安全 | 可能丢失精度（大整数） |
| `double → int` | ⚠️ | 截断小数，可能溢出 |
| `signed → unsigned` | ❌ | 负数变成巨大正数 |
| `int → char` | ⚠️ | 截断高位 |

## 四、const 常量

```cpp
const int N = 100;        // 编译期常量，不可修改
const double PI = 3.14159;

int n;
cin >> n;
const int M = n;          // ⚠️ 运行时常量（C++ 允许，但数组大小不能用它）

int arr[N];               // ✅ N 是编译期常量，可以
// int arr[M];            // ❌ M 是运行时常量，普通数组不行（用 vector）
```

## 五、变量命名规则

- 只能包含字母、数字、下划线，**不能以数字开头**
- 区分大小写（`abc` ≠ `ABC`）
- 不能是 C++ 关键字（`int`, `if`, `for`, `class` ...）
- 不能以双下划线开头（保留给编译器）或下划线+大写字母开头

## ⚠️ 易错点

1. **int 乘法溢出**——`int a=100000,b=100000; a*b` 溢出，必须 `1LL*a*b`。
2. **整型除法截断**——`5/2 == 2`，`double x = 5/2` 结果是 `2.0` 不是 `2.5`。
3. **浮点数不能用 `==` 比较**——用 `fabs(a-b) < EPS`。
4. **signed/unsigned 混用**——`-1 < 1U` 为 false（-1 被转为 UINT_MAX）。
5. **char 的符号性不确定**——C++ 标准未规定 `char` 是 signed 还是 unsigned，取决于编译器（x86 上通常 signed）。
6. **float 字面量必须加 `f`**——`float x = 1.0` 中 `1.0` 是 double，有个隐式转换；`float x = 1.0f` 才是真正的 float。
7. **C++ 中有符号整数溢出是 UB**——不是简单的"回绕"，编译器可能做任何事。
