# 网络流基础

---

## 1. 流网络定义

**流网络** 是一个有向图 G = (V, E)，每条边 (u, v) 有一个非负的**容量** c(u, v)。

网络中有一个**源点 s** 和一个**汇点 t**（s != t）。

**流 f** 是定义在边上满足以下条件的函数：

1. **容量限制**：0 <= f(u, v) <= c(u, v)
2. **流量守恒**：对任意非 s, t 的顶点 u，流入 = 流出

$$\sum_{v \in V} f(v, u) = \sum_{v \in V} f(u, v)$$

---

## 2. 最大流问题

求从 s 到 t 的最大可行流量（即 s 的流出量 = t 的流入量）。

---

## 3. 增广路与残余网络

**残余容量**：`r(u, v) = c(u, v) - f(u, v)`（正向），`r(v, u) = f(u, v)`（反向）。

**增广路**：在残余网络中，从 s 到 t 的一条路径（所有边残余容量 > 0）。

**核心思路**：不断找增广路并推送流量，直到不存在增广路为止。

---

## 4. Edmonds-Karp (EK)

### 核心思想

每次用 **BFS** 找最短的增广路（边数最少），沿路径推送等于路径上最小残余容量的流量。

### 模板

```cpp
const int N = 205, M = 5005;
const int INF = 0x3f3f3f3f;
int n, m, s, t;

struct Edge {
    int to, cap, rev;   // rev: 反向边在 g[to] 中的下标
};
vector<Edge> g[N];
int level[N], iter[N];

void addEdge(int u, int v, int cap) {
    g[u].push_back({v, cap, (int)g[v].size()});
    g[v].push_back({u, 0, (int)g[u].size() - 1});
}

int ek() {
    int maxflow = 0;
    while (true) {
        vector<int> pre(N, -1), preEdge(N, -1);
        queue<int> q;
        q.push(s);
        while (!q.empty()) {
            int u = q.front(); q.pop();
            if (u == t) break;
            for (int i = 0; i < g[u].size(); i++) {
                auto &e = g[u][i];
                if (e.cap > 0 && pre[e.to] == -1) {
                    pre[e.to] = u;
                    preEdge[e.to] = i;
                    q.push(e.to);
                }
            }
        }
        if (pre[t] == -1) break;  // 无增广路

        int flow = INF;
        for (int v = t; v != s; v = pre[v])
            flow = min(flow, g[pre[v]][preEdge[v]].cap);

        for (int v = t; v != s; v = pre[v]) {
            auto &e = g[pre[v]][preEdge[v]];
            e.cap -= flow;
            g[v][e.rev].cap += flow;
        }
        maxflow += flow;
    }
    return maxflow;
}
```

> EK 复杂度 O(V * E^2)，小图够用，但大图推荐 Dinic。

## 5. Dinic 算法

### 核心思想

1. **BFS 分层**：给每个点标上从 s 出发的最短距离 level[u]
2. **DFS 多路增广**：在分层图上用 DFS 同时推送多条增广路
3. **当前弧优化**：用 iter[u] 记录当前处理到哪条边，避免重复遍历

### 模板

```cpp
const int N = 10005, M = 100005;
const int INF = 0x3f3f3f3f;
int n, m, s, t;

struct Edge {
    int to, cap, rev;
};
vector<Edge> g[N];
int level[N], iter[N];

void addEdge(int u, int v, int cap) {
    g[u].push_back({v, cap, (int)g[v].size()});
    g[v].push_back({u, 0, (int)g[u].size() - 1});
}

bool bfs() {
    memset(level, -1, sizeof(level));
    queue<int> q;
    level[s] = 0;
    q.push(s);

    while (!q.empty()) {
        int u = q.front(); q.pop();
        for (auto &e : g[u]) {
            if (e.cap > 0 && level[e.to] < 0) {
                level[e.to] = level[u] + 1;
                q.push(e.to);
            }
        }
    }
    return level[t] >= 0;         // false 表示无增广路
}

int dfs(int u, int t, int f) {
    if (u == t || f == 0) return f;

    for (int &i = iter[u]; i < g[u].size(); i++) {
        Edge &e = g[u][i];
        if (e.cap > 0 && level[u] < level[e.to]) {
            int d = dfs(e.to, t, min(f, e.cap));
            if (d > 0) {
                e.cap -= d;
                g[e.to][e.rev].cap += d;
                return d;
            }
        }
    }
    return 0;                     // 此点无法到达汇点
}

int dinic() {
    int maxflow = 0;
    while (bfs()) {
        memset(iter, 0, sizeof(iter));
        int f;
        while ((f = dfs(s, t, INF)) > 0)
            maxflow += f;
    }
    return maxflow;
}
```

### 复杂度

| 图类型 | Dinic 复杂度 |
|--------|:-----------:|
| 一般图 | O(V^2 * E) |
| 单位容量网络 | O(min(V^{2/3}, E^{1/2}) * E) |
| **二分图匹配** | **O(E * sqrt(V))**，极快 |

## 6. 最小割 = 最大流

**割 (Cut)**：将顶点分为 S（含 s）和 T（含 t）两部分。割的容量 = 所有从 S 到 T 的边容量之和。

**最大流最小割定理**：最大流 = 最小割的容量。

**应用**：求最小割方案：
1. 跑完 Dinic 后
2. 在残余网络上从 s 出发 BFS/DFS，能到达的点放入 S
3. 所有 S->T 的边就是最小割的边集

## 7. 建模技巧

### 7.1 二分图最大匹配

源 s -> 左部 (容量 1) -> 右部 -> 汇 t (容量 1)。Dinic 跑 O(根号V * E)。

```cpp
// nL 个左部点 (1..nL)，nR 个右部点 (nL+1..nL+nR)
s = 0, t = nL + nR + 1;
for (int i = 1; i <= nL; i++) addEdge(s, i, 1);
for (int i = 1; i <= nR; i++) addEdge(nL + i, t, 1);
for (每条原始边 u->v) addEdge(u, nL + v, 1);
```

### 7.2 拆点（限制点的流量）

若题目限制某点的最大通过流量为 limit：
将 u 拆成 u_in 和 u_out，之间连容量为 limit 的边。入边连 u_in，出边连 u_out。

```cpp
// 拆点：原编号 i -> i_in = i, i_out = i + n
for (int i = 1; i <= n; i++)
    addEdge(i, i + n, limit[i]);
```

### 7.3 多源多汇

添加超级源点 SS，连向所有真实源点（容量 INF）；所有真实汇点连向超级汇点 TT（容量 INF）。

## 8. 经典模型

| 模型 | 建图方法 |
|------|----------|
| **二分图最大匹配** | s->左(1)->右(1)->t |

| **最大权闭合子图** | 正权点 s->i，负权点 i->t，原边容量 INF |
| **点限制流量** | 拆点为 in/out |
| **棋盘割** | 黑白染色，s->黑，黑->白，白->t |

## 易错点

1. **反向边容量**：addEdge 中正向 cap，反向 0。增广时正向减、反向加
2. **rev 下标**：用 vector 存边时 rev 存的是反向边在 g[v] 中的下标，不是全局下标
3. **BFS 分层**：level[e.to] < 0 才入队，已访问的点跳过
4. **当前弧优化**：DFS 中 iter[u] 必须用引用 `int &i`，否则优化失效
5. **多次调用**：多组数据时记得清空 g[]、重置迭代器
6. **拆点编号**：u_in = u, u_out = u + n，确保编号不冲突
7. **无向图**：真无向图需要建两条容量均为 cap 的边（不是正向 cap 反向 0）
