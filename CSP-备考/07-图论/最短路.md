# 最短路

---

## 四种算法总览

| 算法 | 类型 | 时间 | 空间 | 负权 | 负环 |
|------|------|------|------|:---:|:---:|
| **Floyd-Warshall** | 多源 | O(n³) | O(n²) | ✅ | ❌ |
| **Dijkstra** | 单源 | O(m log n) | O(n+m) | ❌ | ❌ |
| **Bellman-Ford** | 单源 | O(nm) | O(n+m) | ✅ | ✅ |
| **SPFA** | 单源 | O(m)~O(nm) | O(n+m) | ✅ | ✅ |

> **考场原则**：能写 Dijkstra 就不写 SPFA，n ≤ 300 的多源直接 Floyd。

---

## 1. Floyd-Warshall

### 核心思想

`dp[k][i][j]`：从 i 到 j，只经过前 k 个顶点的最短路径。滚动数组压缩掉第一维：

$$dp[i][j] = \min(dp[i][j],\ dp[i][k] + dp[k][j])$$

三重循环，**k 必须在最外层**。

### 模板

```cpp
const int N = 305;
const int INF = 0x3f3f3f3f;
int dp[N][N];
int n, m;

void floyd() {
    for (int k = 1; k <= n; k++)
        for (int i = 1; i <= n; i++)
            for (int j = 1; j <= n; j++)
                if (dp[i][k] != INF && dp[k][j] != INF)
                    dp[i][j] = min(dp[i][j], dp[i][k] + dp[k][j]);
}

// 初始化
memset(dp, 0x3f, sizeof(dp));
for (int i = 1; i <= n; i++) dp[i][i] = 0;
```

### 扩展

- **传递闭包**：`dp[i][j] |= dp[i][k] && dp[k][j]`（改成 bool）
- **最小环**：在第 k 轮之前，`ans = min(ans, dp[i][j] + g[i][k] + g[k][j])`
- **判负环**：如果 `dp[i][i] < 0`，则存在负环

## 2. Dijkstra

### 核心思想

贪心：每次从未确定最短距离的顶点中，取出距离最小的 u，用 u 松弛所有邻边。

**正确性前提**：所有边权 ≥ 0。因为一旦一个点被"确定"，后续不可能有更短路径。

### 模板（堆优化）

```cpp
const int N = 100005;
typedef pair<int,int> pii;   // {距离, 顶点}
vector<pii> g[N];
int dist[N];
bool vis[N];
int n, m;

void dijkstra(int s) {
    memset(dist, 0x3f, sizeof(dist));
    memset(vis, 0, sizeof(vis));
    priority_queue<pii, vector<pii>, greater<pii>> pq;

    dist[s] = 0;
    pq.push({0, s});

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (vis[u]) continue;          // 关键：懒删除
        vis[u] = true;

        for (auto [v, w] : g[u]) {
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                pq.push({dist[v], v});
            }
        }
    }
}
```

### 为什么不能处理负权？

因为 Dijkstra 的贪心假定"先被确定的点一定是最短的"。如果有负权边，后面经过负边可能更新已经确定的点，导致错误。

> 反例：s→a(1), s→b(2), a→b(-3)。Dijkstra 先确定 a=1，再确定 b=2，但实际 b 最短应该是 -2。

## 3. Bellman-Ford

### 核心思想

对所有边进行 n-1 轮松弛操作。第 k 轮保证从源点出发经过 ≤ k 条边的最短路已被找到。

```cpp
const int N = 100005;
const int INF = 0x3f3f3f3f;
struct Edge {
    int u, v, w;
} e[N];
int dist[N];
int n, m;

bool bellman_ford(int s) {
    memset(dist, 0x3f, sizeof(dist));
    dist[s] = 0;

    for (int i = 1; i < n; i++) {        // n-1 轮
        bool updated = false;
        for (int j = 1; j <= m; j++) {
            if (dist[e[j].u] != INF &&
                dist[e[j].v] > dist[e[j].u] + e[j].w) {
                dist[e[j].v] = dist[e[j].u] + e[j].w;
                updated = true;
            }
        }
        if (!updated) break;             // 提前终止优化
    }

    // 第 n 轮检测负环
    for (int j = 1; j <= m; j++) {
        if (dist[e[j].u] != INF &&
            dist[e[j].v] > dist[e[j].u] + e[j].w) {
            return false;                // 存在负环
        }
    }
    return true;
}
```

> ⚡ 加一个 `updated` 标志，如果某轮没有任何松弛，可以提前退出。

## 4. SPFA (Shortest Path Faster Algorithm)

### 核心思想

Bellman-Ford 的队列优化：只把被松弛过的顶点入队（因为它们的邻边才可能被继续松弛）。

```cpp
const int N = 100005;
vector<pair<int,int>> g[N];
int dist[N], cnt[N];
bool inq[N];
int n, m;

bool spfa(int s) {
    memset(dist, 0x3f, sizeof(dist));
    memset(cnt, 0, sizeof(cnt));
    memset(inq, 0, sizeof(inq));
    queue<int> q;

    dist[s] = 0;
    q.push(s);
    inq[s] = true;
    cnt[s] = 1;

    while (!q.empty()) {
        int u = q.front(); q.pop();
        inq[u] = false;

        for (auto [v, w] : g[u]) {
            if (dist[v] > dist[u] + w) {
                dist[v] = dist[u] + w;
                cnt[v] = cnt[u] + 1;
                if (cnt[v] >= n) return false;  // 负环判定
                if (!inq[v]) {
                    q.push(v);
                    inq[v] = true;
                }
            }
        }
    }
    return true;
}
```

### 负环判定原理

`cnt[v]` 记录从源点到 v 的最短路经过的边数。如果 `cnt[v] ≥ n`，说明经过至少 n 条边到达 v，根据鸽巢原理，必定重复经过某个点 → 有负环。

> ⚠️ **SPFA 容易被卡退化成 O(nm)**，构造数据（网格图 + 特定边序）就能让它 TLE。S 组优先用 Dijkstra + 堆优化或 Bellman-Ford。

## 5. 场景选择速查

| 场景 | 推荐算法 |
|------|----------|
| n ≤ 300，问任意两点最短路 | Floyd |
| 正权图，单源最短路 | Dijkstra (堆优化) |
| 有负权边，问是否存在负环 | Bellman-Ford / SPFA |
| 差分约束系统 | SPFA（建超级源点） |
| 边数较少 (m ≈ n) | SPFA → 被卡就 Bellman-Ford |

## 6. 路径还原

在松弛时记录前驱：

```cpp
int pre[N];   // 前驱数组

// 松弛时：
if (dist[v] > dist[u] + w) {
    dist[v] = dist[u] + w;
    pre[v] = u;
}

// 输出 s → t 的路径：
void printPath(int t) {
    if (pre[t] == t) { cout << t; return; }
    printPath(pre[t]);
    cout << " -> " << t;
}
```

## ⚠️ 易错点

1. **Dijkstra 忘加 vis 判重**：同一顶点可能多次入堆，不加 `if(vis[u]) continue` 会 TLE
2. **Floyd 循环顺序**：k→i→j，k 在最外层！写成 i→j→k 会错
3. **INF 选择**：用 `0x3f3f3f3f`，`memset(dist, 0x3f, sizeof(dist))` 一键初始化
4. **Bellman-Ford 松弛**：必须用上一轮的 dist，不能用本轮更新过的
5. **SPFA 负环判定**：用 `cnt[v] >= n`，不要用 `>= N` 之类的硬编码
