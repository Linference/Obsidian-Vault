# 二分图与匹配

---

## 1. 二分图定义

将顶点分为两个不相交的集合 L 和 R，使得图中**每条边**的端点分别属于 L 和 R。

> 等价条件：图中不含奇数长度的环。

---

## 2. 二分图判定（染色法）

### 核心思想

BFS/DFS 遍历图，相邻顶点染不同颜色（0/1）。若某次发现相邻顶点同色 → 不是二分图。

### 模板（BFS）

```cpp
const int N = 100005;
vector<int> g[N];
int color[N];          // -1 未染色，0/1 两种颜色
int n, m;

bool isBipartite() {
    memset(color, -1, sizeof(color));
    for (int i = 1; i <= n; i++) {
        if (color[i] != -1) continue;
        queue<int> q;
        q.push(i);
        color[i] = 0;
        while (!q.empty()) {
            int u = q.front(); q.pop();
            for (auto v : g[u]) {
                if (color[v] == -1) {
                    color[v] = color[u] ^ 1;    // 交替染色
                    q.push(v);
                } else if (color[v] == color[u]) {
                    return false;                // 冲突！
                }
            }
        }
    }
    return true;
}
```

### DFS 版

```cpp
bool dfs(int u, int c) {
    color[u] = c;
    for (auto v : g[u]) {
        if (color[v] == -1) {
            if (!dfs(v, c ^ 1)) return false;
        } else if (color[v] == c) {
            return false;
        }
    }
    return true;
}
```

## 3. 最大匹配

**匹配**：一组边，任意两条边没有公共顶点。

**最大匹配**：包含边数最多的匹配。

**完美匹配**：所有顶点都被匹配覆盖（仅当 |L| = |R|）。

---

## 4. 匈牙利算法

### 核心思想

**交替路**：从未匹配点出发，依次经过非匹配边、匹配边、非匹配边...的路径。

**增广路**：起点和终点都是**未匹配点**的交替路。将增广路上的匹配边变成非匹配边、非匹配边变成匹配边 → 匹配数 +1。

匈牙利算法本质：对每个左部顶点，尝试找到一条增广路。

### 模板

```cpp
const int N = 505;                    // 左部顶点数
vector<int> g[N];                     // g[u]：u 可达的右部顶点
int matchR[N];                        // matchR[v]：右部 v 匹配的左部顶点
bool vis[N];                          // 每轮 DFS 的访问标记

bool dfs(int u) {
    for (auto v : g[u]) {
        if (vis[v]) continue;
        vis[v] = true;
        if (matchR[v] == 0 || dfs(matchR[v])) {
            matchR[v] = u;
            return true;
        }
    }
    return false;
}

int hungarian() {
    int ans = 0;
    memset(matchR, 0, sizeof(matchR));
    for (int i = 1; i <= nL; i++) {   // nL：左部顶点数
        memset(vis, 0, sizeof(vis));  // 每轮重置 vis
        if (dfs(i)) ans++;
    }
    return ans;
}
```

### 复杂度

O(nL × m)，通常远小于最坏情况。左部 n ≤ 500 时非常安全。

> ⚡ 也可以用最大流求解二分图最大匹配：源→左(容量1)→右→汇(容量1)，Dinic 在二分图上更有效。

## 5. 三个重要定理

| 定理 | 内容 |
|------|------|
| **Kőnig 定理** | 二分图中，**最小点覆盖** = **最大匹配** |
| **独立集定理** | 二分图中，**最大独立集** = n - **最大匹配** |
| **路径覆盖** | DAG 的最小**路径**覆盖 = n - **拆点二分图最大匹配** |

---

## 5.1 最小点覆盖

选出最少的顶点，使得每条边至少有一个端点在选出的集合中。

**构造方法**（基于最大匹配后）：
- 从左侧未匹配点出发沿交替路 DFS
- 左侧无法到达的点 + 右侧可到达的点 = 最小点覆盖

---

## 5.2 最大独立集

选出最多的顶点，任意两点之间没有边。

**公式**：最大独立集 = n - 最小点覆盖 = n - 最大匹配

---

## 5.3 DAG 最小路径覆盖

用最少的**不相交路径**覆盖 DAG 中所有顶点。

**做法**：
1. 拆点：每个 u 拆成 `u_out`（左部）和 `u_in`（右部）
2. 每条边 u → v 建边：`u_out → v_in`
3. 最小路径覆盖 = n - 最大匹配

```cpp
// 路径覆盖建图
for (int u = 1; u <= n; u++) {
    for (auto v : original[u]) {
        g[u].push_back(v);    // 左部 u_out → 右部 v_in
    }
}
int ans = n - hungarian();    // 最小路径覆盖
```

## 7. 棋盘类题目建模技巧

**棋盘二分染色**：按 `(i + j) % 2` 分成两色，相邻格子颜色不同。

```
0 1 0 1
1 0 1 0
0 1 0 1
```

常见转化：
- **棋盘覆盖（骨牌）**：染色的相邻0-1格子连边 → 最大匹配
- **不能攻击的棋子**：攻击范围内的格子连边 → 最大独立集
- **行列匹配**：行→左，列→右，`(i, j)` 位置可放就连边

---

## ⚠️ 易错点

1. **匈牙利 vis 每轮重置**：`memset(vis, 0, sizeof(vis))` 必须放在循环内
2. **matchR 下标**：matchR 数组大小为右部顶点数，下标从 1 开始
3. **二分图判定**：图可能不连通，需对每个未染色点都启动 BFS/DFS
4. **最小路径覆盖**：要求路径**不相交**，若允许相交需先用 Floyd 求传递闭包再建图
5. **Kőnig 定理**：仅在**二分图**成立，一般图不成立
