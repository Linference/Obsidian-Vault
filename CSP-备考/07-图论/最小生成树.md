# 最小生成树 (MST)

---

## 定义

给定无向连通图 G = (V, E)，每条边有权值 w。**最小生成树** 是一棵包含所有 V 的树，使得边权和最小。

> 最小生成树可能不唯一（存在等权边时）。若要求输出方案，需注意唯一性判断。

---

## 1. Kruskal 算法

### 核心思想

**贪心**：将所有边按权值从小到大排序，依次尝试加入每条边。如果边的两端尚未连通（用并查集判断），则加入 MST；否则跳过。

> **正确性**：MST 的割性质——任意割中，最小边必定在某棵 MST 中。Kruskal 正是按此性质贪心。

### 模板

```cpp
const int N = 100005, M = 200005;
int fa[N];

int find(int x) {
    return fa[x] == x ? x : fa[x] = find(fa[x]);
}

struct Edge {
    int u, v, w;
    bool operator<(const Edge &rhs) const {
        return w < rhs.w;
    }
} e[M];

int n, m;

int kruskal() {
    sort(e + 1, e + m + 1);            // 1. 按边权排序
    for (int i = 1; i <= n; i++) fa[i] = i;  // 2. 初始化并查集

    int cnt = 0, ans = 0;              // cnt: 已选边数
    for (int i = 1; i <= m; i++) {
        int fu = find(e[i].u);
        int fv = find(e[i].v);
        if (fu != fv) {                // 3. 不构成环就选
            fa[fu] = fv;
            ans += e[i].w;
            cnt++;
            if (cnt == n - 1) break;   // 4. n-1 条边即完成
        }
    }
    return cnt == n - 1 ? ans : -1;    // -1 表示不连通
}
```

| 复杂度 | O(m log m) | 瓶颈在排序 |
| 适用 | 稀疏图 | 边少点多的场景 |

## 2. Prim 算法

### 核心思想

类似 Dijkstra：从任意起点开始，每次选择连接"已选中顶点集合"和"未选中顶点"的最小权边，将对应顶点加入集合。

### 模板（堆优化版，O(m log n))

```cpp
const int N = 100005;
const int INF = 0x3f3f3f3f;
typedef pair<int,int> pii;

vector<pii> g[N];    // {邻点, 边权}
int dist[N];          // dist[i]: i 到当前 MST 集合的最小边权
bool vis[N];
int n, m;

int prim() {
    memset(dist, 0x3f, sizeof(dist));
    memset(vis, 0, sizeof(vis));
    priority_queue<pii, vector<pii>, greater<pii>> pq;

    dist[1] = 0;
    pq.push({0, 1});
    int ans = 0, cnt = 0;

    while (!pq.empty()) {
        auto [d, u] = pq.top(); pq.pop();
        if (vis[u]) continue;
        vis[u] = true;
        ans += d;
        cnt++;

        for (auto [v, w] : g[u]) {
            if (!vis[v] && dist[v] > w) {    // 注意：和 Dijkstra 的区别！
                dist[v] = w;
                pq.push({dist[v], v});
            }
        }
    }
    return cnt == n ? ans : -1;
}
```

### 朴素版（O(n²)，适合稠密图）

```cpp
int prim_n2() {
    memset(dist, 0x3f, sizeof(dist));
    memset(vis, 0, sizeof(vis));
    dist[1] = 0;
    int ans = 0;

    for (int i = 1; i <= n; i++) {
        int u = -1;
        for (int j = 1; j <= n; j++)          // 找最近的未选中点
            if (!vis[j] && (u == -1 || dist[j] < dist[u]))
                u = j;

        if (dist[u] == INF) return -1;         // 不连通
        vis[u] = true;
        ans += dist[u];

        for (int v = 1; v <= n; v++)           // 松弛邻边
            if (!vis[v] && g[u][v] < dist[v])
                dist[v] = g[u][v];
    }
    return ans;
}
```

## 3. Kruskal vs Prim 对比

| | Kruskal | Prim (堆) | Prim (朴素) |
|------|:-------:|:--------:|:---------:|
| 时间复杂度 | O(m log m) | O(m log n) | O(n²) |
| 适用 | 稀疏图 | 任意 | 稠密图 (m ≈ n²) |
| 代码成本 | 低（需要并查集） | 中 | 低 |
| 能否求次小 | ❌ 不方便 | ✅ 方便 | ✅ |

> 考场上默认用 **Kruskal**，代码短且足以应对 90% 的题目。只有特殊构造（如完全图、n 小 m 极大）才考虑 Prim 朴素版。

## 4. 次小生成树

**定义**：边权和**严格**第二小的生成树（或非严格，允许相同）。

### 算法思路（Kruskal + 倍增）

1. 先用 Kruskal 求出 MST，记录树边和非树边
2. 对 MST 进行 LCA 预处理，同时维护路径上的**最大值**和**严格次大值**
3. 枚举每条非树边 (u, v, w)：
   - 求 MST 上 u→v 路径的最大边权 `mx`
   - 若 `w > mx`，替换后答案 = `MST_ans - mx + w`（严格次小）
   - 若 `w == mx`，用**次大值**替换（严格次小必要条件）
4. 所有替换方案中取最小

> 若图不连通或 n < 3，无次小生成树。

**应用**：判断 MST 唯一性——若次小生成树权值 == MST 权值，则 MST 不唯一。

---

## 5. MST 唯一性判定

1. 按 Kruskal：若存在等权边，其中某条不能加入但另一条同权的可以 → 考虑等权边集合的替换
2. 按次小生成树：`次小 == MST → 不唯一`
3. 更严谨的方法：Kruskal 过程对于每组**等权边**，统计可行边数和实际选择边数，不同则 MST 不唯一

## ⚠️ 易错点

1. **Kruskal 排序索引**：`sort(e+1, e+m+1)`，别忘了 m 是边数而不是 M
2. **Prim 与 Dijkstra 混淆**：Prim 更新 `dist[v] = w`（单条边权），Dijkstra 更新 `dist[v] = dist[u] + w`（路径和）
3. **不连通判断**：MST 必须选满 n-1 条边，否则图不连通
4. **无向图存边**：链式前向星 / vector 存双向边，Kruskal 存边数组只需存一次
5. **并查集初始化**：`for (int i = 1; i <= n; i++) fa[i] = i;`
