# 08-动态规划 MOC

> 动态规划（Dynamic Programming, DP）是 CSP-S 最重要的考点之一，也是竞赛中的核心思维工具。

## 什么是动态规划？

动态规划的核心思想：**将大问题分解为重叠子问题，通过保存子问题的解避免重复计算。**

与分治的区别：DP 的子问题有重叠，分治的子问题相互独立。<br>
与贪心的区别：DP 考虑全局最优，贪心只考虑当前最优。

## DP 解题五步法

1. **确定状态**：确定 `dp[i]` / `dp[i][j]` 的含义
2. **状态转移方程**：如何从子问题推导当前问题
3. **边界条件**：初始值、dp 数组的初值
4. **计算顺序**：确保在计算当前状态时，子问题已经求解
5. **空间优化**：能否用滚动数组压缩？

## 学习路线建议

```
01背包 → 线性DP（LIS/LCS）→ 区间DP → 树形DP → 状压DP → 数位DP → 概率期望 → DP优化
```

- **01背包**是入门，必须彻底理解"逆序枚举"和"滚动数组"
- **线性DP**涵盖最常见的序列问题
- **区间DP**锻炼状态设计的灵活性
- **树形DP**结合树的遍历，是树类问题的核心
- **状压DP**和**数位DP**是进阶技巧，CSP-S 高频
- **DP优化**是冲高分的关键

## 常见误区

- 状态定义不清晰 → 先想清楚状态代表什么再写代码
- 转移方向反了 → 画出状态DAG，确保拓扑序
- 忘记初始化 → dp 数组初值和边界条件同等重要
- 空间爆了 → 先检查是否需要滚动数组
- 循环顺序错误 → 背包问题特别注意正序/逆序
