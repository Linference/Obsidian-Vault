# 哈希表

**哈希表**通过哈希函数将 key 映射到固定范围内的索引，实现平均 O(1) 的增删查。

---

## 1. 哈希函数设计

好的哈希函数要求：**均匀分布、快速计算、冲突少**。

| 类型 | 示例 |
|------|------|
| 除留余数法 | `h(k) = k % mod`，mod 取质数（如 100003） |
| 平方取中法 | 对 k² 取中间几位 |

---

## 2. 冲突处理

### 2.1 拉链法（Chaining）

每个桶是一个链表，冲突时追加到链表尾部。

```cpp
const int MOD = 100003;
vector<int> table[MOD];  // 或用 list<int>

void insert(int x) {
    int h = (x % MOD + MOD) % MOD;  // 处理负数
    table[h].push_back(x);
}

bool find(int x) {
    int h = (x % MOD + MOD) % MOD;
    for (int v : table[h])
        if (v == x) return true;
    return false;
}
```

### 2.2 开放寻址法

冲突时探测下一个位置，直到找到空位。仅用一个数组，但需要 2~3 倍空间。

```cpp
const int MOD = 200003, INF = 0x3f3f3f3f;
int h[MOD];
memset(h, 0x3f, sizeof(h));  // 初始化为 INF 表示空

int find_pos(int x) {
    int k = (x % MOD + MOD) % MOD;
    while (h[k] != INF && h[k] != x) {
        k++;
        if (k == MOD) k = 0;  // 循环
    }
    return k;  // 返回 x 应插入/已存在的位置
}

void insert(int x) { h[find_pos(x)] = x; }
bool query(int x) { return h[find_pos(x)] == x; }
```

## 3. 字符串哈希 ⭐⭐⭐

CSP 最高频的哈希应用。将字符串看作 base 进制数。

```cpp
typedef unsigned long long ull;
const ull BASE = 131;  // 或 13331

ull h[MAXN], p[MAXN];  // h: 前缀哈希, p: base 的幂

void init(const string& s) {
    int n = s.size();
    p[0] = 1;
    for (int i = 1; i <= n; i++) {
        h[i] = h[i - 1] * BASE + s[i - 1];  // 1-indexed
        p[i] = p[i - 1] * BASE;
    }
}

// 子串 s[l..r] 的哈希值 (1-indexed)
ull getHash(int l, int r) {
    return h[r] - h[l - 1] * p[r - l + 1];
}
```

### 为什么用自然溢出（unsigned long long）

`ull` 自动对 `2^64` 取模，速度快且冲突概率极低（除非被卡）。双哈希更安全：

```cpp
const ull BASE1 = 131, MOD1 = 1e9 + 7;
const ull BASE2 = 13331, MOD2 = 1e9 + 9;

// 存储用 pair<ull, ull>
```

### O(1) 子串判重

```cpp
// 判断 s[l1..r1] 和 s[l2..r2] 是否相等
bool equal(int l1, int r1, int l2, int r2) {
    return getHash(l1, r1) == getHash(l2, r2);
}
```

## 4. STL unordered_map / unordered_set

```cpp
#include <unordered_map>
#include <unordered_set>

unordered_map<string, int> mp;
mp["key"] = 1;
if (mp.find("key") != mp.end()) { /* 存在 */ }
mp.count("key");  // 0 或 1

unordered_set<int> st;
st.insert(x); st.erase(x); st.count(x);
```

> ⚠️ 注意：CSP 中 `unordered_map` 可能被出题人构造数据卡成 O(n)。关键比赛建议用 **map**（稳定 O(log n)）或手写哈希。

## 5. 自定义哈希函数

```cpp
// pair<int, int> 的哈希
struct PairHash {
    size_t operator()(const pair<int, int>& p) const {
        return (size_t)p.first * 1000003 + p.second;
    }
};
unordered_map<pair<int, int>, int, PairHash> mp2;
```

## 6. 布隆过滤器（Bloom Filter）简介

一种**概率性**数据结构：用多个哈希函数映射到 bit 数组。判断"可能存在"或"一定不存在"。

- 插入：k 个哈希函数分别置位
- 查询：所有 k 个位都是 1 → "可能存在"；任一为 0 → "一定不存在"
- CSP 不常考，但思维值得了解

---

## ⚠️ 易错点

1. **负数 mod**：`(x % MOD + MOD) % MOD` 才能得到非负数。
2. **字符串哈希 1-indexed**：方便前缀公式 `h[r] - h[l-1] * p[r-l+1]`。
3. **BASE 取 131 而非 26**：避免与小写字母等映射值冲突。
4. **自然溢出可能被卡**：重要比赛准备双哈希。
5. **unordered_map 的 find vs 中括号**：`mp[x]` 不存在时会**插入**默认值，用 `find` 或 `count` 判存在性。
