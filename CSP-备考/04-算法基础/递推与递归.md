# 递推与递归

## 核心思想

**递推（Iterative / DP雏形）**：从已知小规模解，逐步推导大规模解。自底向上，用循环实现。

**递归（Recursion）**：函数调用自身，将大问题分解为同类小问题。自顶向下，用函数调用栈实现。

两者本质上是同一枚硬币的两面，很多递推问题也可以写成递归，反之亦然。

---

## 递推 vs 递归

| | 递推 | 递归 |
|------|------|------|
| 方向 | 自底向上 | 自顶向下 |
| 实现 | 循环 (for/while) | 函数调用自身 |
| 空间 | O(n) 或 O(1)（滚动数组） | O(n)（调用栈深度） |
| 时间 | 无调用开销，快 | 有调用开销，可能重复计算 |
| 栈溢出风险 | 无 | 有（默认栈 8MB，约 2~3 百万层） |
| 代码可读性 | 一般 | 直观，接近数学定义 |

**原则**：能用递推就不用递归。但分治类问题递归更自然。

## 经典递推问题

### 1. 斐波那契数列

F(1)=1, F(2)=1, F(n)=F(n-1)+F(n-2)

```cpp
long long fib(int n) {
    if (n <= 2) return 1;
    long long a = 1, b = 1, c;
    for (int i = 3; i <= n; i++) {
        c = a + b;
        a = b;
        b = c;
    }
    return c;
}
```

滚动数组优化：O(n) 时间，O(1) 空间。对比朴素递归：O(2^n) 时间！

### 2. 走楼梯

N 级台阶，每次走 1 或 2 级，求方案数。

```cpp
int climbStairs(int n) {
    if (n <= 2) return n;
    int a = 1, b = 2, c;
    for (int i = 3; i <= n; i++) {
        c = a + b;
        a = b; b = c;
    }
    return c;
}
```

### 3. 平面分割

n 条直线最多把平面分成几部分？

递推：第 n 条线与前 n-1 条产生 n-1 个交点，新增 n 个区域。
f(n) = f(n-1) + n, f(1) = 2。

```cpp
int maxRegions(int n) {
    int f = 2;
    for (int i = 2; i <= n; i++)
        f += i;
    return f;
}
// 通项公式：f(n) = n*(n+1)/2 + 1
```

## 递归三要素

### 1. 终止条件（Base Case）
必须有，否则无限递归 → 栈溢出。

### 2. 递归关系（Recurrence）
把大问题缩小为同类子问题。

### 3. 向终止条件收敛
参数必须递减，确保到达 base case。

```cpp
// 阶乘：n! = n * (n-1)!
int factorial(int n) {
    if (n == 0) return 1;           // 1. 终止条件
    return n * factorial(n - 1);    // 2+3. 递归关系 + 收敛
}
```

### 调用栈示意图

```
factorial(3)
  -> 3 * factorial(2)
       -> 2 * factorial(1)
            -> 1 * factorial(0)
                 -> return 1
            -> return 1*1 = 1
       -> return 2*1 = 2
  -> return 3*2 = 6
```

每层递归占用栈空间（局部变量 + 返回地址），层数过多 → **栈溢出**。

## 汉诺塔

3 根柱子 A、B、C，A 上有 n 个盘，大小从下到上递减。一次移动一个盘，大盘不能压小盘。把全部盘移到 C。

**递归思路**：
1. 把 n-1 个盘从 A 移到 B（借助 C）
2. 把第 n 个盘从 A 移到 C
3. 把 n-1 个盘从 B 移到 C（借助 A）

```cpp
void hanoi(int n, char from, char aux, char to) {
    if (n == 1) {
        cout << from << " -> " << to << endl;
        return;
    }
    hanoi(n - 1, from, to, aux);
    cout << from << " -> " << to << endl;
    hanoi(n - 1, aux, from, to);
}
```

最少移动次数：2^n - 1，复杂度 O(2^n)。

## 递归转递推

### 斐波那契：递归 -> 记忆化 -> 递推

```cpp
// 1. 朴素递归：O(2^n) —— 大量重复计算
int fib_naive(int n) {
    if (n <= 2) return 1;
    return fib_naive(n - 1) + fib_naive(n - 2);
}

// 2. 记忆化搜索：O(n) —— 用数组存算过的结果
int memo[1000];
int fib_memo(int n) {
    if (n <= 2) return 1;
    if (memo[n]) return memo[n];
    return memo[n] = fib_memo(n - 1) + fib_memo(n - 2);
}

// 3. 递推（滚动数组）：O(n) 时间 + O(1) 空间
int fib_iter(int n) {
    int a = 1, b = 1, c;
    for (int i = 3; i <= n; i++) {
        c = a + b; a = b; b = c;
    }
    return c;
}
```

**转化技巧**：凡是能用记忆化搜索写的递归，都能用数组递推写。只需把递归的"深度"变成数组下标。

## 分治思想

**分治**是递归的重要应用：将问题分解为若干个规模较小的同类子问题，分别求解后合并。

```cpp
// 分治通用框架
void divideAndConquer(int l, int r) {
    if (l >= r) return;            // base case
    int mid = (l + r) / 2;
    divideAndConquer(l, mid);      // 左半
    divideAndConquer(mid + 1, r);  // 右半
    merge(l, mid, r);              // 合并
}
```

三大经典分治算法：

| 算法 | 分解方式 | 合并 | 复杂度 |
|------|---------|------|--------|
| 归并排序 | 对半分 | 合并两个有序数组 | O(n log n) |
| 快速排序 | 按 pivot 分区 | 无需合并 | O(n log n) |
| 二分查找 | 取中点 | 舍弃一半 | O(log n) |

详见 [[排序算法全集]] 和 [[二分查找与二分答案]]。

## 典型真题

### 1. 洛谷 P1044 栈（卡特兰数）
n 个数入栈，问出栈顺序有多少种。
递推公式：f(n) = sum(f(i) * f(n-1-i)) for i=0..n-1，f(0)=1。

### 2. 洛谷 P1002 过河卒
棋盘上 A(0,0) 到 B(n,m)，马控制的点不能走。求路线数。
递推：dp[i][j] = dp[i-1][j] + dp[i][j-1]（可以到达的格子）。

### 3. 洛谷 P1028 数的计算
给定自然数 n，可以每次在左侧加一个 ≤ 一半的数。求能产生多少个数。
递推/记忆化：f(n) = 1 + sum(f(i)) for i=1..n/2。

## ⚠️ 易错点

1. **递归无终止条件**：无限递归 → 段错误（栈溢出）
2. **斐波那契递归复杂度 O(2^n)**：不是 O(n log n) 或 O(n^2)！每个节点分裂两个子问题
3. **递归深度过大爆栈**：一般 10^5~10^6 层就爆了（默认栈 8MB），递归大图 DFS 要小心
4. **递推变量更新顺序错误**：a=b; b=c; 顺序不能反
5. **汉诺塔步骤数用 int 存不下**：n=64 时是 2^64-1，远超 int，需要高精度
6. **分治的 base case 忘记处理**：如归并排序 `if(l>=r) return;` 不写就死循环
7. **记忆化数组未初始化**：memo 要初始化为未计算标记（如 -1 或 0）
