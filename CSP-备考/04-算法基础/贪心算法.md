# 贪心算法

## 核心思想

**贪心算法**：每一步都选择当前看起来最优的决策，期望最终得到全局最优解。

> 贪心 ≠ 直觉。贪心需要**严格证明**，否则很可能出错。

---

## 贪心的证明方法

### 1. 交换论证（Exchange Argument）

假设存在一个更优解，通过**交换相邻元素**，证明交换后不会变差，从而贪心解不劣于任何解。

> 例如：活动选择问题，"选最早结束的活动"可以通过交换论证证明。

### 2. 数学归纳法

证明：若前 k 步贪心选最优，第 k+1 步贪心选仍最优。

### 3. 反证法

假设贪心不是最优，推出矛盾。

## 经典贪心问题

### 1. 活动选择（区间调度）

> N 个活动，每个有开始时间 s[i] 和结束时间 f[i]，同一时间只能参加一个，求最多参加几个。

**贪心策略**：按**结束时间最早**排序，依次选取不和已选冲突的活动。

```cpp
struct Activity { int s, f; };
bool cmp(Activity &a, Activity &b) { return a.f < b.f; }

int maxActivities(Activity a[], int n) {
    sort(a, a + n, cmp);
    int cnt = 1, last = a[0].f;
    for (int i = 1; i < n; i++)
        if (a[i].s >= last) {
            cnt++;
            last = a[i].f;
        }
    return cnt;
}
```

**为什么选最早结束不选最早开始？** 因为最早结束的让出更多剩余空间 → 交换论证可证。

---

### 2. 分数背包（可分割）

> N 个物品，重量 w[i]，价值 v[i]，背包容量 W。物品可以任意分割。

**贪心策略**：按**性价比 v[i]/w[i] 降序**，尽量装性价比高的。

```cpp
struct Item { double w, v; };
bool cmp(Item &a, Item &b) { return a.v / a.w > b.v / b.w; }

double fractionalKnapsack(Item a[], int n, double W) {
    sort(a, a + n, cmp);
    double ans = 0;
    for (int i = 0; i < n && W > 0; i++) {
        double take = min(W, a[i].w);
        ans += take * a[i].v / a[i].w;
        W -= take;
    }
    return ans;
}
```

> **对比**：0-1 背包（不可分割）不能用贪心，必须用 DP！详见 [[06-动态规划]]。

### 3. 合并果子（哈夫曼编码）

> N 堆果子，每次合并任意两堆，代价为两堆重量之和，求总代价最小。

**贪心策略**：每次选最小的两堆合并（优先队列维护）。

```cpp
int mergeFruits(int a[], int n) {
    priority_queue<int, vector<int>, greater<int>> pq;
    for (int i = 0; i < n; i++) pq.push(a[i]);
    int ans = 0;
    while (pq.size() > 1) {
        int x = pq.top(); pq.pop();
        int y = pq.top(); pq.pop();
        ans += x + y;
        pq.push(x + y);
    }
    return ans;
}
```

**证明**：哈夫曼树是最优前缀编码 → 对偶贪心策略的正确性。

### 4. 区间覆盖

> 选最少的区间覆盖整个线段 [0, L]。

**贪心策略**：按**左端点排序**，每次选**能覆盖当前空缺**且**右端点最远**的区间。

```cpp
struct Seg { int l, r; };
bool cmp(Seg &a, Seg &b) { return a.l < b.l; }

int minCover(Seg a[], int n, int L) {
    sort(a, a + n, cmp);
    int cnt = 0, cur = 0, i = 0;
    while (cur < L) {
        int maxR = cur;
        while (i < n && a[i].l <= cur)
            maxR = max(maxR, a[i++].r);
        if (maxR == cur) return -1;   // 无法覆盖
        cur = maxR; cnt++;
    }
    return cnt;
}
```

### 5. 田忌赛马

> 两队马，每局胜者得分，问田忌最高得分。

**贪心策略**：
- 田忌最慢 vs 齐王最慢 → 能赢则赢
- 田忌最快 vs 齐王最快 → 能赢则赢
- 否则，田忌最慢当炮灰消耗齐王最快

```cpp
int tianJiRace(int t[], int q[], int n) {
    sort(t, t + n); sort(q, q + n);
    int tl = 0, tr = n - 1, ql = 0, qr = n - 1, score = 0;
    while (tl <= tr) {
        if (t[tr] > q[qr])      { score++; tr--; qr--; }   // 最快胜
        else if (t[tl] > q[ql]) { score++; tl++; ql++; }   // 最慢胜
        else { if (t[tl] < q[qr]) score--; tl++; qr--; }   // 炮灰
    }
    return score * 200;  // 假设赢+200输-200平0
}
```

## 贪心 vs DP

| | 贪心 | 动态规划 |
|------|------|---------|
| 决策 | 只考虑当前最优 | 考虑所有可能子结构 |
| 复杂度 | O(n log n)（通常要排序） | O(n²) 或更高 |
| 适用范围 | 需满足贪心选择性质 | 满足最优子结构即可 |
| 正确性 | 需严格证明 | 正确（只要状态转移对） |
| 硬币找零 | 人民币(1,5,10,20,50,100)可用贪心 | 任意面额需 DP |

### 贪心翻车案例

找零 15 元，可用硬币面额 {1, 6, 10}。贪心选 10+1+1+1+1+1（6个），最优是 6+6+1+1+1（5个）。因为**没有最优子结构性质**（大的不一定由小的组成）。

## ⚠️ 易错点

1. **不证明就写贪心**：CSP 中贪心题多半需要推理或验证。写之前问自己："为什么不选别的？"
2. **贪心和 DP 混淆**：看到"最大/最小"没想清楚就写贪心 → 大概率 WA
3. **活动选择用"最早开始"策略**：反例 — 一个超长活动覆盖全部 → 只能选 1 个
4. **排序忘记去重 / 边界**：区间覆盖时，多个区间起点相同需要统一处理
5. **浮点数比较**：性价比排序不要直接用 `==` 判等
6. **多组测试数据忘记重置全局变量**
