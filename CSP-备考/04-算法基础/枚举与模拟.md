# 枚举与模拟

## 核心思想

### 枚举（Enumeration）

列举问题解空间中**所有可能的候选解**，逐一验证是否满足条件。暴力但有效，是 CSP-J 的得分基石。

> 能用暴力拿部分分，就绝不空着。

### 模拟（Simulation）

严格按照题目描述的规则和流程，**一步步执行操作**，输出最终结果。不需要数学技巧，考的是**细心和代码能力**。

## 枚举的优化策略

### 1. 缩小枚举范围

```cpp
// 题目：求 x² + y² = n 的正整数解
// 笨办法：枚举 x,y 从 1 到 n → O(n²)
// 聪明法：枚举 x 从 1 到 sqrt(n)，y = sqrt(n - x²)，判整数 → O(√n)

for (int x = 1; x * x <= n; x++) {
    int rest = n - x * x;
    int y = (int)sqrt(rest);
    if (y * y == rest) { /* 找到一组解 */ }
}
```

### 2. 剪枝（提前排除不可能的情况）

```cpp
// 题目：三个数之和为100，求最多方案
int cnt = 0;
for (int a = 0; a <= 100; a++)
    for (int b = 0; b <= 100 - a; b++) {   // b 上限受 a 约束
        int c = 100 - a - b;                 // c 直接算，不用枚举第三层
        cnt++;
    }
```

### 3. 利用对称性

全排列枚举时，只枚举一半，另一半镜像得到。

## 常见枚举模式

### 枚举子集（二进制法）

n 个元素的集合，共 2^n 个子集。用整数的二进制位表示选/不选。

```cpp
int a[] = {1, 2, 3, 4};
int n = 4;
for (int s = 0; s < (1 << n); s++) {       // s 的二进制表示子集
    int sum = 0;
    for (int i = 0; i < n; i++)
        if (s & (1 << i))                   // 第 i 位为 1，选了 a[i]
            sum += a[i];
    // 处理当前子集 sum
}
```

**常用位运算**：

| 操作 | 写法 | 含义 |
|------|------|------|
| 判断第 i 位 | `s & (1 << i)` | 是否选了第 i 个 |
| 置 1 | `s \|= (1 << i)` | 选第 i 个 |
| 置 0 | `s &= ~(1 << i)` | 去掉第 i 个 |
| 翻转 | `s ^= (1 << i)` | 取反 |

### 枚举排列（next_permutation）

```cpp
#include <algorithm>
vector<int> v = {1, 2, 3, 4};
do {
    // 处理当前排列 v
} while (next_permutation(v.begin(), v.end()));
// ⚠️ 必须先排序，否则只生成按字典序大于当前的所有排列
```

## 模拟题的解题框架

```
1. 读题 → 圈出每一步操作的关键词
2. 设计数据结构（数组、矩阵、队列、栈）
3. 用循环/递归逐步推进
4. 边模拟边更新状态
5. 输出最终状态
```

### 真题框架：矩阵变换模拟

```cpp
// 洛谷 P2550 矩阵旋转 — 模拟题模板
int n, m;
int a[105][105], b[105][105];

void rotate() {      // 顺时针旋转 90°
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            b[j][n - 1 - i] = a[i][j];
    swap(n, m);
    // a = b
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            a[i][j] = b[i][j];
}

int main() {
    cin >> n >> m;
    for (int i = 0; i < n; i++)
        for (int j = 0; j < m; j++)
            cin >> a[i][j];
    string ops; cin >> ops;
    for (char op : ops) {
        if (op == 'A') rotate();     // 旋转
        if (op == 'B') flip();       // 翻转
    }
    // 输出
}
```

## 典型真题

### 1. CSP-J 2022 乘方

> 判断 a^b 是否超过 10⁹

```cpp
long long ans = 1;
for (int i = 1; i <= b; i++) {
    ans *= a;
    if (ans > 1e9) { cout << -1; return 0; }
}
cout << ans;
```

**考点**：直接模拟计算过程，边算边判断溢出。

### 2. 洛谷 P1047 校门外的树

> 长度为 L 的马路，M 个区间要移走树，问剩下几棵

```cpp
bool tree[10005];  // true 表示树还在
memset(tree, 1, sizeof(tree));  // 初始全部有树
for (int i = 0; i < M; i++) {
    int l, r; cin >> l >> r;
    for (int j = l; j <= r; j++)
        tree[j] = false;
}
int ans = 0;
for (int i = 0; i <= L; i++)
    if (tree[i]) ans++;
```

**考点**：朴素模拟即可。更优解用[[前缀和与差分|差分]]。

### 3. 洛谷 P2678 跳石头

> 枚举？不，这种最小值最大的题用[[二分查找与二分答案|二分答案]]！

## ⚠️ 易错点

1. **模拟时忘记重置状态**：多组数据必须 memset 或重新初始化
2. **枚举范围过大导致 TLE**：先算一下循环次数，超过 10⁷ 就需要优化
3. **next_permutation 前忘记 sort**：只生成字典序大于当前的所有排列
4. **子集枚举的位运算优先级**：`if (s & (1 << i))` 不加括号会出错（`&` 优先级低于 `==`）
5. **模拟时精度问题**：浮点运算不要用 `==` 判断相等，用 `fabs(a-b) < 1e-9`
6. **多组输入没读到 EOF**：用 `while(cin >> n)` 而非 `cin >> n; if(!n) break`
