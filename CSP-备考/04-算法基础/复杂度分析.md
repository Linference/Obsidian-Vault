# 复杂度分析

## 核心思想

复杂度分析让你在写完代码之前就知道它能不能跑过，是 CSP 最基础也最容易被忽视的能力。两个维度：

- **时间复杂度**：算法执行时间随数据规模 n 的增长趋势
- **空间复杂度**：算法所需额外内存随 n 的增长趋势

我们关心的是 **n→∞ 时的渐近行为**，忽略常数因子和低阶项。用 **大O记号** 表示。

## 大O记号

### 定义

$f(n) = O(g(n))$ 表示存在常数 $C > 0$ 和 $n_0$，使得当 $n \ge n_0$ 时，$f(n) \le C \cdot g(n)$。

通俗：**算法耗时不会超过 g(n) 的常数倍**。

### 三种情况

| 情况 | 含义 | 举例(线性查找) |
|------|------|-----------------|
| 最坏 | 算法保证不超过 | O(n) — 查到末尾 |
| 最好 | 理想输入下的下界 | O(1) — 第一个就是 |
| 平均 | 随机输入的期望 | O(n) — 期望查一半 |

**CSP 中 99% 看最坏复杂度。**

## 常见复杂度排序

$$O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(n^3) < O(2^n) < O(n!)$$

| 复杂度 | n=10 | n=100 | n=10³ | n=10⁵ | n=10⁷ |
|--------|------|-------|-------|-------|-------|
| O(1) | 1 | 1 | 1 | 1 | 1 |
| O(log n) | 3 | 7 | 10 | 17 | 23 |
| O(n) | 10 | 100 | 10³ | 10⁵ | 10⁷ |
| O(n log n) | 30 | 700 | 10⁴ | 1.7×10⁶ | 2.3×10⁸ |
| O(n²) | 100 | 10⁴ | 10⁶ | 10¹⁰ | — |
| O(2^n) | 10³ | 10³⁰ | — | — | — |
| O(n!) | 3.6×10⁶ | 10¹⁵⁸ | — | — | — |

**C++ 一秒约执行 10⁷~10⁸ 次简单操作。** 超过就是 TLE。

## 循环复杂度分析

### 单层循环
```cpp
for (int i = 0; i < n; i++)  // O(n)
  sum += a[i];

for (int i = 0; i < n; i += 2)  // O(n) — 常数倍忽略
  sum += a[i];
```

### 嵌套循环
```cpp
for (int i = 0; i < n; i++)       // 外层 O(n)
  for (int j = 0; j < n; j++)     // 内层 O(n)
    cnt++;                         // 总 O(n²)

for (int i = 0; i < n; i++)       // O(n²) — 内层递减不影响量级
  for (int j = i; j < n; j++)
    cnt++;
```

### 二分/对数
```cpp
while (n > 1) {       // n 每次减半 → O(log n)
  n /= 2;
}

int x = 1;
while (x < n)          // x 每次翻倍 → O(log n)
  x *= 2;
```

### 含 break / continue
最坏复杂度不看 break，要看**最坏输入**下会执行多少次。

## 递归复杂度

### 主定理（Master Theorem）

对于递推式 $T(n) = a \cdot T(n/b) + O(n^d)$：

| 条件 | 复杂度 |
|------|--------|
| $d > \log_b a$ | $T(n) = O(n^d)$ |
| $d = \log_b a$ | $T(n) = O(n^d \log n)$ |
| $d < \log_b a$ | $T(n) = O(n^{\log_b a})$ |

经典应用：
- 归并排序：$T(n) = 2T(n/2) + O(n)$ → $O(n \log n)$
- 二分查找：$T(n) = T(n/2) + O(1)$ → $O(\log n)$
- 斐波那契递归：$T(n) = T(n-1) + T(n-2) + O(1)$ → $O(2^n)$ ⚠️

## 空间复杂度

程序所占用的额外空间（不包括输入数据本身）。

```cpp
int a[n];           // O(n)
int dp[n][n];       // O(n²)
vector<int> v;      // O(k) — 实际元素数

// 递归的空间 = 调用栈深度 × 每层空间
int fib(int n) { return fib(n-1) + fib(n-2); }  // 栈深 O(n)，时间 O(2^n)
```

## 数据范围反推算法（核心技能）

**已知 n 的范围，猜出题人意图：**

| n 的范围 | 可能复杂度 | 常见算法 |
|----------|-----------|---------|
| n ≤ 10 | O(n!) | 全排列枚举、搜索 |
| n ≤ 20 | O(2^n) | 状态压缩DP、子集枚举 |
| n ≤ 100 | O(n³) | Floyd、区间DP |
| n ≤ 500 | O(n³) | 三重循环DP |
| n ≤ 1000 | O(n²) | 简单DP、冒泡/选择排序 |
| n ≤ 10⁵ | O(n log n) | 排序、二分、线段树 |
| n ≤ 10⁶ | O(n log n) | 贪心、双指针、前缀和 |
| n ≤ 10⁷ | O(n) | 线性筛、桶排序 |
| n ≤ 10⁹ | O(log n) 或 O(1) | 二分答案、数学公式 |

> **经验法则**：看到 `n ≤ 10⁵`，想到 O(n log n) 或 O(n)；看到 `n ≤ 10³`，多半是 O(n²)。

## 真题场景

- **CSP-J 2020 直播获奖**：n=10⁵ → 需要 O(n log n)，用优先队列维护前60%
- **CSP-S 2021 廊桥分配**：n=10⁵ → 需要 O(n log n)，用 set+贪心
- **洛谷 P2249 查找**：q=10⁶ 次查询 → O(log n) 二分

## ⚠️ 易错点

1. **把 O(n²) 当成 O(2^n)**：嵌套循环是 n²，不是指数
2. **忽略 string 操作的复杂度**：`s += c` 均摊 O(1)，`s = s + c` 是 O(n)
3. **STL 容器操作的复杂度**：`set::find` O(log n)，`unordered_set::find` 均摊 O(1)
4. **递归空间也算空间复杂度**：深搜可能爆栈（一般栈 8MB）
5. **不要把常数优化当成复杂度优化**：`i++` vs `i+=2` 不会改变 O(n)
6. **排序后遍历 = O(n log n + n) = O(n log n)**，不是 O(n)
