# 剪枝与记忆化搜索

## 剪枝 (Pruning)

### 核心思想

在 DFS 搜索过程中，**提前判断当前分支不可能产生比已知更优的解**，直接"剪掉"，不再深入。好的剪枝能把指数级复杂度降到可接受范围。

### 五种经典剪枝

#### 1. 可行性剪枝

当前状态已经不满足约束条件，立即返回。

```cpp
// 例：选数之和不能超过 target
void dfs(int i, int sum) {
    if (sum > target) return;       // 可行性剪枝：已经超了
    if (i == n) {
        if (sum == target) ans++;
        return;
    }
    // ...
}
```

#### 2. 最优性剪枝

当前值已经不可能优于已知最优解，立即返回。

```cpp
int best = INF;

void dfs(int i, int cur) {
    if (cur >= best) return;       // 最优性剪枝：不可能更优
    if (到达终点) {
        best = min(best, cur);
        return;
    }
    // ...
}
```

#### 3. 顺序剪枝（改变搜索顺序）

优先搜索"更可能产生解"的分支，让最优性剪枝更早触发。

- 排序后从大到小搜索 → 更快达到 target
- 先搜可能性大的方向 → 更早找到解，剪掉更多分支

```cpp
// 小木棍：从长到短搜，更快触发剪枝
sort(sticks, sticks + n, greater<int>());
```

#### 4. 排除等效冗余

避免搜索本质上相同的状态。

```cpp
// 组合枚举（不重复）：规定后面选的数 >= 前面选的数
void dfs(int start, int sum) {
    if (sum == target) { ans++; return; }
    for (int i = start; i < n; i++) {   // 从 start 开始，不往回选
        if (sum + a[i] > target) break; // 可行性剪枝
        dfs(i + 1, sum + a[i]);         // 下一个从 i+1 开始
    }
}
```

#### 5. 预处理剪枝

搜索前预先排除不可能的情况。

## 记忆化搜索 (Memoization)

### 核心思想

**记忆化搜索 = DFS + DP**。把 DFS 的中间结果缓存起来，避免重复计算。本质是**自上而下 (Top-Down)** 的动态规划。

```cpp
int memo[STATE_SIZE];
memset(memo, -1, sizeof(memo));

int dfs(状态) {
    if (memo[状态] != -1) return memo[状态];    // 查缓存

    if (边界条件) return memo[状态] = 基础值;

    int res = 0;   // 或 INF / -INF
    for (每个子状态) {
        res = 最优(res, dfs(子状态) + 代价);
    }
    return memo[状态] = res;                     // 存缓存
}
```

### 经典例题：数字三角形

```cpp
int n;
int a[105][105];
int memo[105][105];

int dfs(int i, int j) {
    if (i == n) return a[i][j];
    if (memo[i][j] != -1) return memo[i][j];
    return memo[i][j] = a[i][j] + max(dfs(i + 1, j), dfs(i + 1, j + 1));
}
```

## 经典例题：小木棍 (Sticks)

**题目**：给出 n 根小木棍的长度，拼成若干根等长的原始木棍，求最小的原始木棍长度。

**全搜索是 O(n!)，必须用剪枝才能过。**

```cpp
#include <bits/stdc++.h>
using namespace std;

int n, a[70], targetLen, totalLen;
bool used[70];

// 参数：已拼好的原始木棍数，当前正在拼的木棍的已拼长度，上一次选择的木棍下标
bool dfs(int cnt, int cur, int last) {
    if (cnt * targetLen == totalLen) return true;   // 全部拼完
    if (cur == targetLen)                           // 当前这根拼完
        return dfs(cnt + 1, 0, 0);                  // 开始拼下一根

    for (int i = last; i < n; i++) {
        if (used[i]) continue;
        if (cur + a[i] > targetLen) continue;        // 可行性剪枝

        used[i] = true;
        if (dfs(cnt, cur + a[i], i + 1))
            return true;
        used[i] = false;

        // ⚠️ 核心剪枝（跳过等效冗余）
        // 剪枝1：如果当前是第一根或最后一根就失败，直接返回
        if (cur == 0 || cur + a[i] == targetLen)
            return false;
        // 剪枝2：跳过相同长度的木棍
        while (i + 1 < n && a[i + 1] == a[i]) i++;
    }
    return false;
}

int solve() {
    sort(a, a + n, greater<int>());            // 从大到小排序
    for (int len = a[0]; len <= totalLen; len++) {
        if (totalLen % len != 0) continue;
        targetLen = len;
        memset(used, 0, sizeof(used));
        if (dfs(0, 0, 0)) return len;
    }
    return totalLen;
}
```

### 小木棍剪枝总结

1. **从大到小排序** → 更快达到目标长度
2. **`cur == 0` 剪枝**：一根新木棍的第一个小木棍就放不进去 → 这个 targetLen 不行
3. **`cur + a[i] == targetLen` 剪枝**：刚好拼满时失败 → 这个 targetLen 不行
4. **跳过相同长度**：同一位置尝试相同长度的不同木棍是冗余的

## 数位 DP（记忆化搜索实现）

记忆化搜索是数位 DP 最自然的写法。

```cpp
// 统计 [1, n] 中数位和 = target 的数的个数
int digit[20], len;
int memo[20][200][2];   // pos, sum, tight(是否受限)

int dfs(int pos, int sum, bool tight) {
    if (pos == len) return sum == target;
    if (memo[pos][sum][tight] != -1) return memo[pos][sum][tight];

    int limit = tight ? digit[pos] : 9;
    int res = 0;
    for (int d = 0; d <= limit; d++) {
        if (sum + d > target) break;               // 可行性剪枝
        res += dfs(pos + 1, sum + d, tight && (d == limit));
    }
    return memo[pos][sum][tight] = res;
}
```

## 复杂度比较

| 方法 | 不剪枝 | 合理剪枝 | 记忆化 |
|------|--------|----------|--------|
| 全排列 | O(n!) | — | N/A |
| 小木棍 | O(n!) | 能过 n≤64 | N/A |
| 方格路径 | O(2ⁿ) | — | O(n²) |
| 数位 DP | O(10ⁿ) | — | O(n·状态) |

## CSP 真题场景

- **J2020 T4** 方格取数 — 记忆化搜索 / DP
- **S2019 T2** 括号树 — DFS + 记忆化
- **S2018 T2** 货币系统 — 可用记忆化搜索
- 小木棍是经典搜索剪枝模板题，在各类模拟赛中高频出现

## ⚠️ 易错点

1. **memo 初始化**：用 `-1` 标记"未计算"，不要用 `0`（可能 0 是有效结果）
2. **记忆化的维度**：所有影响结果的变量都要纳入状态（包括 `tight` 这种限制位）
3. **剪枝过于激进**：剪掉本该有的解 → 答案偏大或偏小
4. **顺序剪枝的方向**：先大后小 vs 先小后大，不同题目方向不同
5. **跳过相同长度时**：`while` 循环的条件要确保 `i+1 < n`
