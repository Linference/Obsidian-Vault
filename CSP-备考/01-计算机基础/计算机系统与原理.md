## 2. 冯·诺依曼体系结构 ⭐⭐⭐

### 五大部件

| 部件 | 英文 | 功能 |
|------|------|------|
| **运算器** | ALU | 算术运算 + 逻辑运算 |
| **控制器** | CU | 取指令 → 分析 → 执行 |
| **存储器** | Memory | 存储程序和数据 |
| **输入设备** | Input | 数据输入 |
| **输出设备** | Output | 结果输出 |

```
输入设备 → 存储器 ←→ 运算器
              ↕
          控制器 → 输出设备
```

### 核心思想：存储程序
- 程序和数据以**二进制**形式存放在存储器中
- 按地址顺序取出指令执行

---

## 3. CPU（中央处理器）⭐⭐

CPU = **运算器（ALU）** + **控制器（CU）** + **寄存器**

| 部件 | 作用 |
|------|------|
| ALU | 算术（+ - × ÷）和逻辑（与或非）运算 |
| CU | 指令译码、时序控制 |
| 寄存器 | CPU 内部高速存储（如 PC、IR、ACC） |

> **性能指标**：主频（Hz）、字长（bit）、核心数

---

## 4. 存储器层次 ⭐⭐⭐

```
速度 ↑          容量 ↑
寄存器 (CPU内)     最小 (字节级)
↕ L1 Cache
↕ L2 Cache
↕ L3 Cache
↕ 主存 RAM         几 GB
↕ 外存 (硬盘/SSD)  几百 GB ~ TB
```

| 类型 | 特点 | 易失性 |
|------|------|--------|
| **Cache** | SRAM，CPU 片内，极快 | 断电丢失 |
| **RAM**（内存） | DRAM，插在主板上 | **断电丢失** |
| **ROM** | 只读，存 BIOS | **断电不丢** |
| **硬盘/SSD** | 外存，持久化 | 断电不丢 |

### 常见考点

| 问题 | 答案 |
|------|------|
| RAM vs ROM 区别？ | RAM 可读写/断电丢失，ROM 只读/断电不丢 |
| Cache 的作用？ | 解决 CPU 和内存速度不匹配 |
| 存储速度排序？ | 寄存器 > Cache > RAM > SSD > HDD |
| 1KB = ? B | **1024B**（不是 1000！） |
| 1MB = ? KB | 1024KB |
| 1GB = ? MB | 1024MB |

## 6. 总线（Bus）

| 总线 | 作用 | 方向 |
|------|------|------|
| **数据总线** | 传输数据 | 双向 |
| **地址总线** | 传输地址 | 单向（CPU→） |
| **控制总线** | 传输控制信号 | — |

> 地址总线宽度决定最大寻址空间：n 位 → 2^n 个地址

## 7. 软件分类

| 类别 | 说明 | 例子 |
|------|------|------|
| **系统软件** | 管理硬件、提供平台 | OS、编译器、驱动程序 |
| **应用软件** | 完成特定任务 | Word、浏览器、游戏 |

> ⚠️ 编译器和操作系统都属于**系统软件**，这是常见陷阱！

## 8. 编程语言分类

| 类别 | 说明 | 例子 |
|------|------|------|
| **机器语言** | 二进制指令，CPU 直接执行 | 010101... |
| **汇编语言** | 助记符，一一对应机器指令 | `MOV AX, BX` |
| **高级语言** | 接近自然语言，需编译/解释 | C++、Python |

| 执行方式 | 说明 | 代表语言 |
|---------|------|---------|
| **编译型** | 先编译成机器码再执行 | C、C++、Pascal |
| **解释型** | 边解释边执行 | Python、JavaScript、BASIC |
