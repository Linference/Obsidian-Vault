# 计算机网络基础

## 一、网络分层模型

### OSI 七层模型（理论标准）

从下到上：

| 层号 | 名称 | 功能 | 常见协议/设备 |
|:---:|------|------|---------------|
| 7 | **应用层** | 为用户程序提供网络服务 | HTTP, FTP, SMTP, DNS |
| 6 | 表示层 | 数据格式转换、加解密 | SSL/TLS |
| 5 | 会话层 | 建立、管理、终止会话 | RPC |
| 4 | **传输层** | 端到端可靠/不可靠传输 | TCP, UDP |
| 3 | **网络层** | 路由选择、逻辑寻址 | IP, ICMP, ARP, 路由器 |
| 2 | **数据链路层** | 帧封装、物理寻址（MAC） | MAC, 交换机/网桥 |
| 1 | 物理层 | 比特流传输（电压/光纤/无线电） | 集线器、网线、光纤 |

> **记忆口诀**：**应表会传网数物**（从顶到底）。

### TCP/IP 四层模型（事实标准）

| TCP/IP 层 | 对应 OSI | 关键协议 |
|-----------|:-------:|----------|
| **应用层** | 5~7 | HTTP, HTTPS, FTP, DNS, SMTP, POP3 |
| **传输层** | 4 | TCP, UDP |
| **网络层** | 3 | IP (IPv4/IPv6), ICMP, ARP |
| **网络接口层** | 1~2 | 以太网, Wi-Fi, PPP |

> **CSP 常考**：OSI 七层 vs TCP/IP 四层的对应关系。TCP 在哪一层 → 传输层。HTTP 在哪一层 → 应用层。

## 二、各层协议详解

### 应用层

| 协议 | 全称 | 端口 | 传输协议 | 用途 |
|------|------|:---:|:-------:|------|
| **HTTP** | 超文本传输协议 | **80** | TCP | 网页浏览（明文） |
| **HTTPS** | HTTP + SSL/TLS | **443** | TCP | 安全网页浏览（加密） |
| **FTP** | 文件传输协议 | **21**(控制) / 20(数据) | TCP | 文件上传下载 |
| **SSH** | 安全外壳协议 | **22** | TCP | 远程安全登录 |
| **SMTP** | 简单邮件传输协议 | 25 | TCP | 发送邮件 |
| **POP3** | 邮局协议 v3 | 110 | TCP | 接收邮件 |
| **DNS** | 域名系统 | **53** | **TCP/UDP** | 域名→IP 解析 |
| **DHCP** | 动态主机配置协议 | 67/68 | UDP | 自动分配 IP |

> **必背端口**：80(HTTP), 443(HTTPS), 21(FTP), 22(SSH), 53(DNS)。选一个记一个，选择题白送分。

### 传输层

| 特性 | TCP | UDP |
|------|-----|-----|
| 连接方式 | **面向连接**（三次握手） | **无连接**（直接发包） |
| 可靠性 | **可靠**（确认+重传） | 不可靠（尽力而为） |
| 顺序性 | 有序 | 无序 |
| 速度 | 慢 | 快 |
| 适用场景 | 网页、文件传输、邮件 | 视频直播、DNS、在线游戏 |

> **三次握手**：SYN → SYN+ACK → ACK。四次挥手：FIN → ACK → FIN → ACK。

### 网络层

**IP 协议**：负责将数据包从源地址路由到目的地址，提供"尽力而为"的无连接服务。

**ARP（地址解析协议）**：已知 IP 地址，查询对应的 MAC 地址。在同一局域网内广播 ARP 请求实现。

## 三、IP 地址

### IPv4 地址

32 位二进制，分 4 段，每段 8 位，点分十进制表示（如 `192.168.1.1`）。

### IP 地址分类

| 类别 | 最高位 | 网络号范围 | 默认子网掩码 | 用途 |
|:---:|:------:|------------|-------------|------|
| **A 类** | `0` | 1~126 | `255.0.0.0` | 大型网络 |
| **B 类** | `10` | 128~191 | `255.255.0.0` | 中型网络 |
| **C 类** | `110` | 192~223 | `255.255.255.0` | 小型局域网 |
| D 类 | `1110` | 224~239 | — | 多播 |
| E 类 | `1111` | 240~255 | — | 保留 |

> `127.x.x.x` 是**回环地址（Loopback）**，`127.0.0.1` = localhost（指本机）。

### 子网掩码

子网掩码用于判断两个 IP 是否在同一子网：`IP1 & 掩码 == IP2 & 掩码` → 同一子网，可直接通信。

```cpp
// 判断逻辑
bool sameSubnet(uint32_t ip1, uint32_t ip2, uint32_t mask) {
    return (ip1 & mask) == (ip2 & mask);
}
```

> **真题**：给定 IP `192.168.1.100/24`，"24"表示子网掩码前 24 位为 1，即 `255.255.255.0`。

### 私有地址（内网地址）

| 类别 | 私有地址范围 |
|:---:|-------------|
| A | `10.0.0.0` ~ `10.255.255.255` |
| B | `172.16.0.0` ~ `172.31.255.255` |
| C | `192.168.0.0` ~ `192.168.255.255` |

> 私有地址不能直接在公网路由，需要 NAT（网络地址转换）。

### IPv6

- **128 位**（IPv4 的 4 倍长度），分 8 组，每组 4 位十六进制，如 `2001:0db8:85a3::8a2e:0370:7334`
- 解决了 IPv4 地址枯竭问题（$2^{128} \approx 3.4\times 10^{38}$ 个地址）
- `::` 表示连续的 0 段可省略（但只能用一次）
- `::1` = IPv6 的回环地址

## 四、域名解析（DNS）

```
用户输入 www.example.com
  → 浏览器查本地 DNS 缓存
  → 查操作系统 hosts 文件
  → 向 DNS 服务器发起查询（UDP 53端口）
  → 递归查询：根DNS → .com 顶级DNS → example.com 权威DNS
  → 返回 IP 地址（如 93.184.216.34）
  → 浏览器用该 IP 建立 TCP 连接
```

## ⚠️ 易错点

1. **OSI 七层 vs TCP/IP 四层**——千万别搞混。"网络层"在 OSI 是第 3 层，在 TCP/IP 也叫网络层。
2. **TCP 是面向连接的可靠协议，UDP 是无连接的不可靠协议**——但 UDP 也有用武之地（实时性要求高）。
3. **DNS 用 UDP（默认），区域传输用 TCP**——不要一概而论说 DNS 只用 UDP。
4. **127.0.0.1 是回环地址**，不是 A 类网络中的普通地址。
5. **子网掩码 255.255.255.0 即 /24**——/24 表示前 24 位为 1。
6. **IPv4 是 32 位，IPv6 是 128 位**——一个是 $2^{32}$，一个是 $2^{128}$，数量级天差地别。
7. **MAC 地址 48 位（6 字节）**——与 IP 不同，MAC 是物理地址，出厂即固定（可修改但一般不）。
