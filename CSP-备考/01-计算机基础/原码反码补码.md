# 原码反码补码

## 一、为什么需要补码

计算机只会做加法。如果直接用"符号位 + 数值位"的原码做减法，电路会变得极其复杂——需要额外设计减法器、处理借位、处理正零和负零。

**补码（Two's Complement）** 的发明解决了一切：**用加法实现减法**。$A - B = A + (-B)$，一台加法器搞定所有运算。

> 这是计算机体系结构中最漂亮的数学设计之一。

## 二、三种编码的定义

以下假设 8 位二进制（1 字节），最高位为符号位（0 正 1 负）。

### 原码（Sign-Magnitude）

直接给数值前面加符号位。

| 数值  | 原码               |
| :-: | ---------------- |
| +5  | `00000101`       |
| -5  | `10000101`       |
| +0  | `00000000`       |
| -0  | `10000000`（两个零！） |

**缺点**：两个零；做减法需要特殊处理。

### 反码（Ones' Complement）

正数反码 = 原码。负数反码 = 符号位不变，其余位按位取反。

| 数值 | 原码 | 反码 |
|:---:|------|------|
| +5 | `00000101` | `00000101` |
| -5 | `10000101` | `11111010` |
| +0 | `00000000` | `00000000` |
| -0 | `10000000` | `11111111`（仍有两个零！） |

### 补码（Two's Complement）

正数补码 = 原码。负数补码 = **反码 + 1**。

| 数值 | 原码 | 补码 |
|:---:|------|------|
| +5 | `00000101` | `00000101` |
| -5 | `10000101` | `11111011` |
| +0 | `00000000` | `00000000` |
| -0 | `10000000` | `00000000`（唯一零！） |

> **关键**：补码只有 **一个零**。`-0` 的补码经过"反码+1"后溢出变成 `00000000`，解决了双零问题。

## 三、补码的范围

### 8 位补码

- 最大正数：`01111111` = **+127**
- 最大负数：`10000000` = **-128**（反码+1 溢出到这个值）
- 范围：**-128 ~ +127**，共 256 个数（含 0）

### n 位补码范围公式

$$-2^{n-1} \le \text{值} \le 2^{n-1} - 1$$

| 位数 | 最小值 | 最大值 |
|:---:|:------:|:------:|
| 8 位 | -128 | +127 |
| 16 位 | -32768 | +32767 |
| 32 位 | -2147483648 | +2147483647 |
| 64 位 | -9.22×10^18 | +9.22×10^18 |

> **注意**：负数范围比正数多一个！因为 0 占了一个正数位置。

## 四、补码运算与溢出

### 加法示例

```
  (+5) 00000101
+ (-3) 11111101  ← -3 的补码
-----------------
  (+2) 00000010  ← 进位溢出丢弃（这就是补码的精妙！）
```

### 溢出判断

**溢出**：运算结果超出了给定位数能表示的范围。

三种溢出判断方法：

1. **符号位法**：两个正数相加得负数 → 溢出；两个负数相加得正数 → 溢出。
2. **进位法**：最高位的进位 ⊕ 次高位的进位 = 1 → 溢出。
3. **范围法**：检查结果是否在合法范围内。

```cpp
// 经典溢出陷阱
int a = 2147483647;  // INT_MAX
a = a + 1;           // 溢出！a = -2147483648 (INT_MIN)
// 未定义行为？对于有符号整数是 UB！
```

### 为什么补码能实现 A - B = A + (-B)

$A - B = A + (2^n - B)$。$2^n - B$ 恰好等于 $(\sim B) + 1$（B 的补码）。模 $2^n$ 运算下，减去 B 等价于加上 $2^n - B$。这是模运算的数学性质。

## 五、C++ 中的补码相关陷阱

```cpp
int x = -2147483648;  // INT_MIN
int y = -x;           // ⚠️ 溢出！INT_MIN 没有对应的正数 (2147483648 > INT_MAX)

int a = 0;
unsigned int b = 0;
cout << ~a;  // -1（有符号）
cout << ~b;  // 4294967295（无符号，= UINT_MAX）

int c = -1;
unsigned int d = 1;
if (c < d) { ... }  // ⚠️ false！因为 -1 被隐式转换为 UINT_MAX
```

> **S 组高频**：有符号与无符号比较时的隐式转换陷阱。

## ⚠️ 易错点

1. **8 位补码最小是 -128 不是 -127**——很多人会漏掉这个"多出来的负数"。
2. **INT_MIN 取绝对值会溢出**——`abs(INT_MIN)` 在 32 位下 = INT_MIN 本身（未定义行为）。
3. **`~1 == -2`，不是 0**——按位取反得到的是补码表示的负数。
4. **有符号和无符号比较**——有符号数会被隐式转换为无符号数，导致 -1 > 所有正数。
5. **补码的符号扩展**——将 8 位有符号数转为 16 位时，高位用符号位填充（`11111011` → `11111111 11111011`），无符号数填 0。
6. **C++ 中有符号整数溢出是未定义行为（UB）**——编译器可能做任何事，不能依赖溢出回绕。
