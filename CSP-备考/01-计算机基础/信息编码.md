# 信息编码

## 一、什么是编码

计算机只能存储和处理 0/1 的二进制数据。**编码（Encoding）** 就是把文字、符号、图片、声音等信息映射为二进制数字的规则。

### 为什么需要了解编码

CSP 初赛年年考"ASCII 码值""汉字在 UTF-8 中占几个字节""某某编码是否兼容 ASCII"。纯记忆题，记对=拿分，记错=白丢。

## 二、ASCII 码

**ASCII（American Standard Code for Information Interchange）** 是最基础的字符编码标准，使用 7 位二进制（1 字节中的低 7 位），共 128 个字符。

### 关键码值（必背）

| 字符 | 十进制 | 十六进制 | 助记 |
|:----:|:------:|:--------:|------|
| `'0'` | **48** | 0x30 | 数字起于 48 |
| `'A'` | **65** | 0x41 | 大写起于 65 |
| `'a'` | **97** | 0x61 | 小写起于 97 |
| 空格 | 32 | 0x20 | |
| 回车 CR | 13 | 0x0D | `\r` |
| 换行 LF | 10 | 0x0A | `\n` |
| DEL | 127 | 0x7F | 最大 ASCII 值 |

### 规则记忆

- 数字 < 大写字母 < 小写字母
- 大写字母和小写字母相差 **32**（`'a' - 'A' = 32`）→ 大小写转换：`ch ^= 32` 或 `ch += 32`
- 同一个字母大小写之间：`'a' - 'A' = 32 = 0b100000` → 修改第 6 位即可切换

```cpp
char c = 'A';
c |= 32;   // 'A' → 'a'   （将第6位置1）
c &= ~32;  // 'a' → 'A'   （将第6位清零）
c ^= 32;   // 'A' ↔ 'a'   （翻转第6位）
```

### 真题常考

- "ASCII 码用几位二进制表示？" → 7 位（但存储占 1 字节 = 8 位）
- "'A' 的 ASCII 码是多少？" → 65
- "已知 'D' 的码值，求 'H' 的码值" → 每差一个字母差 1

## 三、Unicode 与 UTF-8

### Unicode

ASCII 只能表示英文，中文、日文、阿拉伯文等无法表示。**Unicode** 为全世界每个字符分配唯一编码（码点 Code Point），目前已收录超过 14 万个字符。

### UTF-8：变长编码方案

UTF-8 是最流行的 Unicode 实现方式，**兼容 ASCII**（前 128 个字符与 ASCII 完全一致，也是 1 字节）。

| 码点范围 | UTF-8 字节数 | 字节格式 |
|----------|:-----------:|----------|
| U+0000 ~ U+007F | **1 字节** | `0xxxxxxx` |
| U+0080 ~ U+07FF | **2 字节** | `110xxxxx 10xxxxxx` |
| U+0800 ~ U+FFFF | **3 字节** | `1110xxxx 10xxxxxx 10xxxxxx` |
| U+10000 ~ U+10FFFF | **4 字节** | `11110xxx 10xxxxxx 10xxxxxx 10xxxxxx` |

> **必记**：常见汉字（基本多文种平面 BMP 内）在 UTF-8 中占 **3 字节**。中文标点（如"。"）也占 3 字节。

### 为什么 UTF-8 在 Web 中广泛使用

1. 兼容 ASCII，英文文本与传统系统无缝对接
2. 变长编码，英文节约空间
3. 无字节序（Endianness）问题

## 四、汉字编码

中国汉字编码经历了三个阶段：

| 标准 | 全称 | 收录字符数 | 编码长度 | 特点 |
|------|------|:---------:|:--------:|------|
| **GB2312** | 信息交换用汉字编码字符集·基本集 | 6763 汉字 | 2 字节 | 1980年，最基础 |
| **GBK** | 汉字内码扩展规范 | 21886 汉字 | 2 字节 | 兼容 GB2312，增繁体/生僻字 |
| **GB18030** | 信息交换用汉字编码字符集 | 70244 汉字 | 1/2/4 字节可变 | 现行国标，兼容 GBK/GB2312 |

> **CSP 常考**：GB2312 中一个汉字占 2 字节。UTF-8 中一个汉字占 3 字节。两者不直接兼容（但 GB18030 兼容 Unicode）。

### 编码识别技巧

- 一个文件如果用 GBK 打开是乱码，换 UTF-8 可能正常——说明原文件是 UTF-8。
- `0xEF 0xBB 0xBF`（BOM）出现在文件头表示 UTF-8 编码。

## 五、常见编码对比

| 编码 | 英文 | 中文 | 兼容 ASCII | 变长 |
|------|:---:|:---:|:---------:|:---:|
| ASCII | 1B | ❌ | ✅ | ❌ |
| GB2312 | 1B | 2B | ✅ | ❌ |
| GBK | 1B | 2B | ✅ | ❌ |
| UTF-8 | 1B | 3B | ✅ | ✅ |
| UTF-16 | 2B | 2B 或 4B | ❌ | ✅ |
| UTF-32 | 4B | 4B | ❌ | ❌ |

## ⚠️ 易错点

1. **ASCII 是 7 位**，不是 8 位——虽然有 8 位存储，但只用低 7 位，最高位为 0。
2. **'0'=48，不是 0**——字符 '0' 的值是 48，数字 0 的值才是 0。`int('0') == 48`。
3. **UTF-8 中英文 1 字节、中文 3 字节**——不要记成"UTF-8 一律 3 字节"。
4. **GBK 中一个汉字 2 字节**——sizeof("中") 在 GBK 环境 = 3（2 字节 + '\0'），在 UTF-8 环境 = 4（3 字节 + '\0'）。
5. **大小写差 32**，不是 26——很容易和字母表顺序搞混。
